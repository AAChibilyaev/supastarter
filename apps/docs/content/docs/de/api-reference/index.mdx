# API-Referenz

Willkommen bei der AACsearch v1 REST API-Referenz. Alle Endpunkte erfordern eine Bearer-Token-Authentifizierung mit einem API-Schlüssel.

## Authentifizierung

Alle API-Anfragen müssen einen `Authorization: Bearer ***`-Header enthalten. API-Schlüssel werden über das Dashboard oder die [Keys-API](/api-reference/keys/keyid) erstellt.

## Endpunktgruppen

- **Projekte** — Organisationen/Projekte verwalten
- **Indexe** — Suchindexe erstellen und verwalten
- **Dokumente** — Dokumente indizieren, durchsuchen und verwalten
- **Suche** — Suchanfragen ausführen und Vorschläge erhalten
- **Analysen** — Suchanalysen und Nutzungsdaten abrufen
- **Schlüssel** — API-Schlüssel verwalten
- **Synonyme** — Suchsynonyme konfigurieren
- **Kurationen** — Suchergebnis-Übersteuerungen konfigurieren
- **Sortierung** — Sortierbare Felder verwalten
- **Facetten** — Facettenfelder verwalten

Probieren Sie den [Erste Schritte](/getting-started/overview)-Leitfaden für eine Einführung oder durchsuchen Sie die Endpunktgruppen in der Seitenleiste.
