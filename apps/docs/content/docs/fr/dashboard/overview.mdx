Le tableau de bord AACsearch est l'application SaaS protégée dans `apps/saas`. Il est organisé en deux portées : **au niveau de l'organisation** (votre espace de travail de recherche) et **au niveau du compte** (votre compte personnel et les outils d'administration).

## Portées du tableau de bord

### Portée organisation (`/[orgSlug]/...`)

Tout ce qui concerne la recherche pour une organisation spécifique se trouve ici. Une organisation correspond à un locataire dans AACsearch — tous les indexes, clés API, analyses et facturations lui sont limités.

| Chemin                        | Zone                                                                            |
| ----------------------------- | ------------------------------------------------------------------------------- |
| `/[orgSlug]/overview`         | Tableau de bord KPI, requêtes principales, fil d'activité, alertes de quota     |
| `/[orgSlug]/getting-started`  | Liste de contrôle d'intégration en 6 étapes                                     |
| `/[orgSlug]/search`           | Espace de travail de recherche (onglets Indexes, Playground, API Keys, Widget)  |
| `/[orgSlug]/api-keys`         | Clés API sur tous les indexes                                                   |
| `/[orgSlug]/import-jobs`      | Historique des jobs d'import                                                    |
| `/[orgSlug]/preview`          | Aperçu / terrain de jeu de recherche                                            |
| `/[orgSlug]/analytics`        | Analyses de recherche — CTR, requêtes principales, sessions                     |
| `/[orgSlug]/relevance`        | Synonymes, curations, mots vides, préréglages                                   |
| `/[orgSlug]/connectors`       | Statut du module CMS, tokens connecteur, historique des jobs de synchronisation |
| `/[orgSlug]/knowledge`        | Espaces de connaissance (RAG/GraphRAG — surface produit séparée)                |
| `/[orgSlug]/settings/general` | Paramètres de l'organisation                                                    |
| `/[orgSlug]/settings/members` | Gestion des membres, invitations                                                |
| `/[orgSlug]/settings/billing` | Facturation et plan de l'organisation                                           |

### Portée compte (`/...`)

Les paramètres personnels et les outils d'administration se trouvent en dehors du contexte d'organisation.

| Chemin                         | Zone                                                                              |
| ------------------------------ | --------------------------------------------------------------------------------- |
| `/settings/general`            | Profil du compte                                                                  |
| `/settings/security`           | Mot de passe, 2FA (TOTP, passkeys), connexions OAuth                              |
| `/settings/notifications`      | Préférences de notification                                                       |
| `/settings/billing`            | Facturation personnelle                                                           |
| `/settings/billing/ai-credits` | Wallet IA (développement actif — v0.6)                                            |
| `/knowledge`                   | Espaces de connaissance personnels                                                |
| `/chatbot`                     | Chatbot IA                                                                        |
| `/admin/*`                     | Outils d'administration du site (utilisateurs admin uniquement — voir ci-dessous) |

## La page Vue d'ensemble

La page de vue d'ensemble de l'organisation (`/[orgSlug]/overview`) est la page de destination principale après la sélection d'une organisation. Elle affiche :

### Tuiles KPI

Quatre métriques résumées pour la période sélectionnée (24h / 7j / 30j) :

| Tuile                           | Source                                                       |
| ------------------------------- | ------------------------------------------------------------ |
| Total des recherches            | Consommation d'unités de recherche des habilitations du plan |
| Documents indexés               | Nombre de documents des habilitations du plan                |
| Utilisation du quota            | `searches.percentUsed` depuis `entitlements.plan`            |
| Jobs de synchronisation échoués | Dérivé du statut du pipeline d'ingestion                     |

Une bannière d'avertissement apparaît lorsque le quota atteint 80%, et une bannière d'erreur lorsqu'il est dépassé.

### Recherches dans le temps

Un graphique sparkline montrant le volume de recherche sur la période sélectionnée. Les données proviennent des lignes `SearchUsageEvent` via la procédure oRPC `search.usage`.

### Requêtes principales

Une liste classée des 10 requêtes de recherche les plus fréquentes pour la période. Alimentée par la procédure `search.topQueries` qui regroupe les lignes `SearchUsageEvent` par texte de requête.

### Fil d'activité

Une liste chronologique des événements récents dérivée des horodatages de création d'index et des comptages de clés API. Une procédure `recentActivity` dédiée est prévue pour v1.x ; le fil actuel est dérivé des données de requête existantes.

### Statut du connecteur

Indique si un connecteur a appelé le point de terminaison de signal de vie récemment. Le statut est dérivé de `lastUsedAt` sur les tokens connecteur :

- **En ligne** — dernier signal de vie dans les 5 dernières minutes
- **Inconnu** — dernier signal de vie dans les 30 dernières minutes
- **Hors ligne** — pas de signal de vie récent

### Carte du plan

Affiche le nom et le statut du plan actif (actif / annulé / en retard de paiement). Pointe vers la page de facturation.

## Sélecteur de période

La page de vue d'ensemble supporte un sélecteur de période (24h / 7j / 30j) stocké dans l'URL sous la forme `?period=7d`. Cela contrôle toutes les agrégations KPI sur la page.

## Structure de navigation

La barre latérale comporte 9 éléments au niveau de l'organisation :

1. Vue d'ensemble
2. Démarrage
3. Search
4. Analyses
5. Pertinence
6. Connecteurs
7. Knowledge
8. Paramètres
9. (Facturation — imbriqué sous Paramètres)
