# Руководство по миграции: v1 → v2

Это руководство поможет вам перейти с AACsearch v1 REST API на v2. API v2 представляет собой значительный шаг вперед с улучшенными возможностями поиска, новыми endpoints, автоматически генерируемыми SDK и улучшенным опытом разработчика.

> **Статус v2**: В активной разработке. Смотрите [Версионирование и устаревание API](/ru/api-reference/versioning-deprecation) для текущего статуса жизненного цикла.

## График

| Веха        | Дата    | Влияние                                             |
| ----------- | ------- | --------------------------------------------------- |
| v2 Beta     | Q3 2026 | Ранний доступ для избранных клиентов                |
| v2 GA       | Q4 2026 | Общая доступность, v1 переходит в статус deprecated |
| v1 Sunset   | Q2 2027 | v1 endpoints начинают возвращать `410 Gone`         |
| v1 Удаление | Q3 2027 | v1 API полностью выводится из эксплуатации          |

Все даты являются приблизительными. Подпишитесь на [Changelog](/ru/changelog) для объявлений.

## Что нового в v2

### Изменения в API

| Область            | v1                                     | v2                                                                          |
| ------------------ | -------------------------------------- | --------------------------------------------------------------------------- |
| **Базовый путь**   | `/api/v1/...`                          | `/api/v2/...`                                                               |
| **Поиск**          | Базовый проход Typesense               | Улучшенный поиск с гибридным ранжированием (семантический + ключевые слова) |
| **Аутентификация** | Bearer Token (ключ API)                | Bearer Token + OAuth 2.0 Client Credentials                                 |
| **Формат ошибок**  | `{ "error": "...", "message": "..." }` | Расширен с `requestId`, `details`, `documentationUrl`                       |
| **SDK**            | Написанные вручную                     | Автоматически генерируемые из OpenAPI 3.1                                   |
| **Пагинация**      | `page` / `perPage`                     | Курсорная через `after` / `before`                                          |

## Критические изменения

### Обязательные изменения

| Изменение         | Действие по миграции                                                               |
| ----------------- | ---------------------------------------------------------------------------------- |
| **Базовый URL**   | Замените `/api/v1/` на `/api/v2/` во всех путях запросов                           |
| **Формат ошибок** | Обновите парсеры ошибок — v2 добавляет `requestId`, `details` и `documentationUrl` |
| **Пагинация**     | Замените `page`/`perPage` на курсорные параметры `after`/`before`                  |

## Пошаговая миграция

### Шаг 1: Проверьте использование API

```bash
rg 'api/v1/' .
rg 'ss_search_' .
```

### Шаг 2: Обновите базовые URL

```diff
- https://api.aacsearch.com/api/v1/indexes/:indexId/search
+ https://api.aacsearch.com/api/v2/indexes/:indexId/search
```

### Шаг 3: Обновите обработку ошибок

```typescript
interface V2Error {
	error: string;
	message: string;
	requestId: string; // НОВОЕ — ID для поддержки
	details?: Record<string, unknown>; // НОВОЕ — ошибки валидации
	documentationUrl?: string; // НОВОЕ — ссылка на документацию
}
```

### Шаг 4: Мигрируйте пагинацию

```diff
- GET /api/v1/indexes/:indexId/documents?page=2&perPage=20
+ GET /api/v2/indexes/:indexId/documents?after=eyJpZCI6InNvbWVJRCJ9&limit=20
```

### Шаг 5: Обновите SDK

```diff
- npm install @aacsearch/client
+ npm install @aacsearch/js-sdk
```

## Поддержка миграции

- **Окно миграции**: v1 остается полностью поддерживаемой в течение **6 месяцев** после v2 GA
- **Staging-среда**: Тестируйте миграцию на `https://staging-api.aacsearch.com/api/v2/`
- **Changelog**: Следите за [Changelog](/ru/changelog) для объявлений, связанных с миграцией
