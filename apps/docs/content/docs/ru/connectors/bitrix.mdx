> **Статус: Ранний просмотр.** Скелет модуля (`aac.search`) функционален — рукопожатие, полная синхронизация, дельта-синхронизация через обработчики событий iblock и фоновая синхронизация через агенты Bitrix работают. Упаковка установки и полировка UI администратора ещё в процессе. Bitrix24 **облачный** (REST/OAuth) — отдельный трек и пока не поддерживается.

Модуль AACsearch для Bitrix (`modules/bitrix/aac.search`) интегрирует ваш каталог 1C-Bitrix self-hosted с AACsearch. Он обеспечивает:

- Экспорт товаров из iblock/catalog Bitrix в AACsearch через Connector API
- Запуск дельта-синхронизации через обработчики событий iblock
- Выполнение фоновой синхронизации через агенты Bitrix (аналог cron-заданий в Bitrix)
- Встраивание хостингового виджета поиска

## Требования

- 1C-Bitrix 24+ (self-hosted, on-premise)
- Включённый модуль `catalog` Bitrix
- PHP 7.4 или выше с `Bitrix\Main\Web\HttpClient`
- Аккаунт AACsearch с хотя бы одним поисковым индексом
- Токен коннектора (`ss_connector_*`), привязанный к этому индексу

> **Bitrix24 облачный** (REST API, OAuth, `catalog.catalog.list`) — отдельный трек коннектора и не включён в данный модуль.

## Структура файлов модуля

```
modules/bitrix/aac.search/
  install/
    index.php           # Установщик/деинсталлятор модуля
    version.php         # Информация о версии модуля
    step_install.php    # Шаг мастера установки
    step_uninstall.php  # Шаг мастера деинсталляции
  lib/
    Client.php          # HTTP-клиент, оборачивающий вызовы Connector API
    ProductExporter.php # Нормализует товары iblock Bitrix → структура ProductDocument
    SyncAgent.php       # Агент Bitrix для фоновой/периодической синхронизации
    EventHandlers.php   # Слушатели событий iblock для дельта-синхронизации
  admin/
    aac_search_settings.php    # Страница администрирования настроек
    aac_search_diagnostics.php # Страница администрирования диагностики
  options.php                  # Начальная загрузка параметров модуля
```

## Шаги установки

### 1. Загрузите модуль

Скопируйте директорию `modules/bitrix/aac.search/` в `/local/modules/aac.search/` на вашем сервере Bitrix. Затем перейдите в **Администрирование Bitrix → Marketplace → Список модулей** и установите `aac.search`.

### 2. Настройте модуль

Откройте **Администрирование → aac.search → Настройки** и заполните:

| Поле               | Описание                                                                |
| ------------------ | ----------------------------------------------------------------------- |
| URL API AACsearch  | Базовый URL вашего экземпляра AACsearch (например, `https://app.aacsearch.com`) |
| Project ID         | ID вашей организации из панели управления AACsearch                     |
| Токен коннектора   | Токен `ss_connector_*`, созданный в Панели управления → Connectors      |
| IBlock / каталог   | Выберите iblock(s), содержащий ваш каталог товаров                      |
| Тип цены           | Какой тип цены Bitrix экспортировать                                    |
| Валюта             | Код валюты для полей цены                                               |
| Интервал синхронизации | Минуты между запусками агента фоновой синхронизации               |
| Режим виджета      | `auto-inject`, `component` или `manual` (см. раздел Виджет ниже)        |
| Режим отладки      | Включить подробное логирование                                          |

### 3. Проверьте соединение

Нажмите **Проверить соединение** на странице настроек. Это вызывает `POST /api/connectors/handshake`. Успешный ответ показывает слаг подключённого индекса.

### 4. Запустите полную синхронизацию

Нажмите **Полная синхронизация каталога** на странице настроек. Модуль:

1. Читает все товары из выбранных iblock-ов с помощью `CIBlockElement::GetList`
2. Нормализует каждый товар с помощью `ProductExporter`
3. Отправляет товары пакетами на `POST /api/projects/:projectId/sync/full`
4. Опрашивает `GET /api/projects/:projectId/sync/jobs/:jobId` для получения статуса
5. Отображает результаты на странице диагностики

## Структура ProductDocument

`ProductExporter` сопоставляет поля iblock Bitrix со стандартной структурой документа:

```json
{
	"external_id": "1234",
	"title": "Blue Running Shoes",
	"description": "Lightweight running shoes",
	"sku": "BRS-42",
	"brand": "Acme Sport",
	"categories": ["Shoes", "Running"],
	"category_ids": ["10", "22"],
	"price": 5999.0,
	"sale_price": 4799.0,
	"currency": "RUB",
	"image_url": "https://myshop.example.ru/img/brs-42.jpg",
	"product_url": "https://myshop.example.ru/catalog/brs-42/",
	"availability": "in_stock",
	"stock_quantity": 42,
	"attributes": { "color": "blue", "size": "42" },
	"locale": "ru"
}
```

## Обработчики событий

Модуль регистрирует обработчики событий iblock для автоматической дельта-синхронизации:

| Событие                      | Триггер                                                         |
| ---------------------------- | --------------------------------------------------------------- |
| `OnAfterIBlockElementAdd`    | Новый товар — запускает дельта-синхронизацию                    |
| `OnAfterIBlockElementUpdate` | Товар обновлён — запускает дельта-синхронизацию                 |
| `OnAfterIBlockElementDelete` | Товар удалён — отправляет запрос на удаление                    |
| `OnPriceUpdate`              | Изменена цена (требует модуль catalog Bitrix)                   |
| `OnIBlockSectionUpdate`      | Обновлён раздел/категория                                       |

## Фоновый агент

Модуль регистрирует агент Bitrix для периодической синхронизации и дренирования буфера дельта:

```php
\Aac\Search\SyncAgent::run();
```

Интервал агента настраивается в параметрах модуля (по умолчанию: каждые 15 минут). Используйте для:

- Повтора неудавшихся элементов дельта-синхронизации
- Периодической полной синхронизации, если обработчики событий пропустили события (например, массовые импорты)

## Размещение виджета

Доступны три режима:

### Автоматическое встраивание (Auto-inject)

Модуль автоматически добавляет тег `<script>` виджета в секцию `<head>` вашего шаблона:

```html
<script
	src="https://app.aacsearch.com/api/widget/widget.js"
	data-base-url="https://app.aacsearch.com"
	data-api-key="ss_search_***"
	data-index-slug="products"
	data-container="#aac-search"
	data-theme="auto"
></script>
```

### Компонент Bitrix

Разместите компонент AACsearch вручную в вашем шаблоне:

```php
<?$APPLICATION->IncludeComponent("aac:search.widget", "", []);?>
```

### Ручное размещение

Скопируйте сниппет из панели управления AACsearch и вставьте его в шаблон напрямую. На странице настроек модуля отображается текущий сниппет для вашего индекса.

Значение `data-api-key` должно быть ключом `ss_search_*`, а не токеном коннектора. Создайте ключ поиска в Панели управления → Search → API Keys.

## Устранение неполадок

**Проверка соединения не удалась**
Убедитесь, что токен коннектора действителен и не отозван. Проверьте, включено ли расширение PHP `curl`. Убедитесь, что URL API AACsearch доступен с вашего сервера.

**Товары не появляются после полной синхронизации**
Индексирование асинхронно. Подождите 30–60 секунд. Проверьте Панель управления → Overview на наличие статуса конвейера приёма данных и неудавшихся пакетов.

**Обработчики событий не срабатывают**
Убедитесь, что модуль установлен и обработчики событий зарегистрированы. В администрировании Bitrix проверьте список событий в разделе **Настройки → Системные настройки → Обработчики событий**.

**Агент не запускается**
Убедитесь, что планировщик агентов Bitrix активен. Проверьте **Администрирование → Настройки → Производительность → Агенты**, чтобы убедиться, что `Aac\Search\SyncAgent::run()` указан и запущен.
