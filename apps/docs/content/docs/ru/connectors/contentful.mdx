> **Статус: Интеграция на основе вебхуков.** В отличие от серверных модулей (PrestaShop, Bitrix), Contentful подключается к AACsearch через исходящие вебхуки. Устанавливать пакет не требуется — вы настраиваете вебхуки в панели управления Contentful, указывая на AACsearch Connector API.

Коннектор AACsearch для Contentful использует вебхуки Contentful для синхронизации ваших записей контента с AACsearch. Он обеспечивает:

- Экспорт записей в AACsearch через Connector API
- Запуск дельта-синхронизации при событиях публикации/снятия с публикации/удаления записей
- Обработку URL-адресов изображений через Contentful Images API
- Синхронизацию контента с учётом локали (встроенная локализация Contentful)

## Требования

- Пространство Contentful с опубликованными записями
- Как минимум одна модель контента, настроенная для индексации поиска
- Аккаунт AACsearch с хотя бы одним созданным поисковым индексом
- Токен коннектора (`ss_connector_*`), привязанный к этому индексу
- Ключ API поиска (`ss_search_*`) для виджета

## Настройка вебхука Contentful

### 1. Создание вебхука в Contentful

Перейдите в **Settings → Webhooks** в вашем пространстве Contentful и нажмите **Add Webhook**.

Настройте следующие поля:

| Поле         | Значение                                                             |
| ------------ | -------------------------------------------------------------------- |
| Имя          | `AACsearch Sync`                                                     |
| URL          | `https://app.aacsearch.com/api/projects/{projectId}/sync/contentful` |
| Метод        | `POST`                                                               |
| Тип контента | `application/json`                                                   |
| Триггер      | **Конкретные события** (см. ниже)                                    |

### 2. Настройка событий-триггеров

Выберите следующие события:

- **Entry** → `publish`, `unpublish`, `delete`
- **Asset** → `publish` (для обновления URL-адресов изображений)

Опционально можно включить **Entry** → `save` для синхронизации в реальном времени во время редактирования, но это значительно увеличит объём вебхуков.

### 3. Добавление пользовательских заголовков

Добавьте следующие заголовки для аутентификации:

| Заголовок       | Значение                    |
| --------------- | --------------------------- |
| `Authorization` | `Bearer ss_connector_{...}` |
| `X-AAC-Project` | Ваш ID проекта AACsearch    |

### 4. Фильтрация по типу контента (рекомендуется)

В разделе **Filters** добавьте фильтр типа контента, чтобы вебхук срабатывал только для моделей, которые вы хотите индексировать:

| Поле         | Значение  |
| ------------ | --------- |
| Тип контента | `product` |
| Тип контента | `article` |

Вы можете добавить один фильтр на каждый индексируемый тип контента. Записи всех остальных типов будут игнорироваться.

## Сопоставление модели контента

Записи сопоставляются с документами AACsearch с помощью правил сопоставления полей. Стандартное сопоставление выглядит следующим образом:

| Поле Contentful      | Поле документа AACsearch | Примечания                                |
| -------------------- | ------------------------ | ----------------------------------------- |
| `sys.id`             | `external_id`            | Уникальный идентификатор из Contentful    |
| `fields.title`       | `title`                  | Основной заголовок записи                 |
| `fields.description` | `description`            | Rich text или обычный текст               |
| `fields.slug`        | `url`                    | URL-дружественный путь                    |
| `fields.body`        | `content`                | Полное содержимое                         |
| `fields.category`    | `categories`             | Массив названий категорий                 |
| `fields.tags`        | `tags`                   | Массив тегов                              |
| `fields.image`       | `image_url`              | URL ресурса (см. обработку изображений)   |
| `fields.price`       | `price`                  | Числовое значение (если применимо)        |
| `fields.locale`      | `locale`                 | Локаль записи (см. синхронизацию локалей) |

### Пользовательское сопоставление полей

Если ваша модель контента использует другие имена полей, настройте сопоставление в payload вебхука или используйте трансформацию полей AACsearch Connector API:

```json
{
	"field_mapping": {
		"external_id": "sys.id",
		"title": "fields.headline",
		"description": "fields.teaser",
		"content": "fields.body_text"
	}
}
```

## Синхронизация с учётом локалей

Contentful имеет встроенную локализацию — каждая запись может иметь значения на нескольких языках. Коннектор AACsearch обрабатывает это следующим образом:

- Каждый языковой вариант записи индексируется как **отдельный документ** с собственным значением `locale`
- К `external_id` добавляется суффикс с кодом локали (например, `abc123_en`, `abc123_ru`)
- Все записи учитывают конфигурацию локали индекса AACsearch
- Если поле пусто для данной локали, используется значение запасной локали (по умолчанию Contentful)

### Конфигурация локалей

Настройте поведение синхронизации локалей через параметры запроса в URL вебхука:

| Параметр        | По умолчанию | Описание                                                                                      |
| --------------- | ------------ | --------------------------------------------------------------------------------------------- |
| `locales`       | `*`          | Список локалей через запятую (например, `en,ru`). `*` синхронизирует все локали пространства. |
| `defaultLocale` | `en-US`      | Локаль Contentful по умолчанию, используемая для запасных вариантов                           |

Пример с фильтрацией локалей:

```
https://app.aacsearch.com/api/projects/{projectId}/sync/contentful?locales=en,ru&defaultLocale=en-US
```

## Обработка URL-адресов изображений

Contentful предоставляет ресурсы через свой API изображений (`images.ctfassets.net`). Коннектор автоматически разрешает ссылки на ресурсы:

1. Когда запись содержит ссылку на ресурс Contentful в поле `image`, коннектор получает `sys.id` ресурса
2. Он формирует URL изображения Contentful: `https://images.ctfassets.net/{spaceId}/{assetId}/{fileName}`
3. URL устанавливается как `image_url` в документе AACsearch
4. Если событие `publish` включено для ресурсов, коннектор повторно синхронизирует затронутые записи при изменении изображений

Вы можете дополнительно применять трансформации изображений с помощью параметров Contentful Images API:

```
https://images.ctfassets.net/{spaceId}/{assetId}/{fileName}?w=800&h=600&fit=fill
```

Настройте параметры изображений через запрос вебхука коннектора:

```
https://app.aacsearch.com/api/projects/{projectId}/sync/contentful?image_params=w%3D800%2Ch%3D600%2Cfit%3Dfill
```

## Формат payload вебхука

Contentful отправляет payload вебхуков в стандартном формате. AACsearch Connector API обрабатывает их и извлекает соответствующие данные. Пример payload для события публикации записи:

```json
{
	"sys": {
		"type": "Entry",
		"id": "abc123def",
		"contentType": {
			"sys": { "id": "product" }
		},
		"version": 5
	},
	"fields": {
		"title": {
			"en-US": "Wireless Headphones",
			"ru": "Беспроводные наушники"
		},
		"description": {
			"en-US": "Premium wireless headphones with noise cancellation",
			"ru": "Премиальные беспроводные наушники с шумоподавлением"
		},
		"slug": {
			"en-US": "wireless-headphones"
		},
		"price": {
			"en-US": 199.99
		}
	}
}
```

Для событий удаления коннектор извлекает `sys.id` из payload и удаляет соответствующие документы из индекса.

## Встраивание виджета

После синхронизации записей встройте виджет AACsearch на ваш сайт. Contentful по умолчанию не имеет серверного рендеринга, поэтому виджет добавляется на стороне клиента:

1. Добавьте элемент-контейнер в шаблон страницы:

```html
<div id="aac-search"></div>
```

2. Загрузите скрипт виджета в `<head>` страницы или перед закрывающим тегом `</body>`:

```html
<script
	src="https://app.aacsearch.com/api/widget/widget.js"
	data-base-url="https://app.aacsearch.com"
	data-api-key="ss_search_***"
	data-index-slug="contentful-entries"
	data-container="#aac-search"
	data-theme="auto"
></script>
```

Значение `data-api-key` — это ключ `ss_search_*`, а не токен коннектора. Этот поисковый ключ может быть только для чтения и безопасен для встраивания в браузер.

Для фреймворков вроде Next.js или Gatsby, которые используют Contentful как headless CMS, вы можете встроить виджет через компонент `<Script>` или пользовательский элемент `<Head>`.

## Устранение неполадок

**Вебхук возвращает 401 Unauthorized**
Проверьте, что заголовок `Authorization` использует действительный токен `ss_connector_*`. Токены можно создать и отозвать в панели управления AACsearch в разделе Коннекторы.

**Записи синхронизируются, но не отображаются в поиске**
Синхронизация помещает записи в очередь `SearchIngestBuffer`; индексация происходит асинхронно. Подождите 30–60 секунд и проверьте поиск. Проверьте статус конвейера приёма данных в Панели управления → Обзор.

**Поля rich text отображаются как сырой JSON**
Contentful отправляет rich text в виде дерева узлов JSON (формат Mobiledoc/TextKit). Настройте коннектор для извлечения обычного текста или HTML из структуры rich text. Установите параметр `rich_text_format` в URL вебхука:

```
https://app.aacsearch.com/api/projects/{projectId}/sync/contentful?rich_text_format=html
```

Поддерживаемые значения: `raw` (по умолчанию, отправляет дерево JSON), `text` (извлечение обычного текста), `html` (HTML-рендеринг).

**Изображения не отображаются в результатах поиска**
Убедитесь, что событие вебхука `publish` для ресурсов включено и что записи ресурсов имеют правильные имена файлов. Если установлены параметры изображений, убедитесь, что строка запроса правильно закодирована в URL.

**Виджет не отображается на странице**
Убедитесь, что элемент-контейнер `#aac-search` существует на странице. Проверьте, что `data-api-key` — это ключ `ss_search_*`, а не токен коннектора. Проверьте консоль браузера на наличие ошибок загрузки скрипта.
