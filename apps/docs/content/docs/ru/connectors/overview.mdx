AACsearch подключается к вашей существующей CMS через две взаимодополняющие поверхности:

- **Connector API** — набор HTTP-эндпоинтов, которые вызывают модули CMS для отправки данных о товарах в AACsearch. Модуль работает на вашем сервере; AACsearch никогда не опрашивает и не парсит ваш магазин.
- **Хостинговый виджет** — Vanilla JS-бандл, обслуживаемый AACsearch по адресу `/api/widget/widget.js`. Вы вставляете один тег `<script>` в тему вашей витрины, и виджет берёт на себя UI поиска, фасеты, сортировку и отслеживание кликов.

Обе поверхности используют разные типы токенов. Ни одна из них никогда не раскрывает административные учётные данные Typesense.

## Модель токенов коннектора

Токены коннектора используют префикс `ss_connector_*` и хранятся как строки `SearchApiKey` с областью `connector_write`. Они создаются в панели управления AACsearch и привязаны к конкретному поисковому индексу.

Свойства токена:

- Префикс: `ss_connector_`
- Область: `connector_write`
- Хранится как: SHA-256 хэш (формат prefix:hash) в `SearchApiKey.hash`
- Открытый текст возвращается один раз при создании — больше никогда
- Можно отозвать в любое время из панели управления

Создайте токен коннектора в **Панели управления → Connectors → New Connector Token** или через процедуру oRPC `search.createConnectorToken`.

## Поддерживаемые модули CMS

| Платформа               | Статус                    | Примечания                                                                             |
| ----------------------- | ------------------------- | -------------------------------------------------------------------------------------- |
| Strapi 5.x              | Альфа                     | npm-плагин с lifecycle hooks; [документация](/docs/connectors/strapi)                  |
| Contentful              | Руководство по интеграции | Синхронизация через вебхуки (без плагина); [документация](/docs/connectors/contentful) |
| Sanity.io               | Руководство по интеграции | GROQ-экспорт + слушатель реального времени; [документация](/docs/connectors/sanity)    |
| PrestaShop 8.x          | Ранний просмотр           | Скелет PHP-модуля существует; отдельный трек коммитов                                  |
| 1C-Bitrix (self-hosted) | Ранний просмотр           | Скелет PHP-модуля существует; отдельный трек коммитов                                  |
| Прямой API              | Доступно                  | Любая система может вызывать Connector API с токеном                                   |

Плагин Strapi — это npm-пакет, устанавливаемый непосредственно в проект Strapi, предоставляющий lifecycle hooks, маппинг типов контента и панель настроек администратора. Для Contentful и Sanity.io AACsearch интегрируется на уровне вебхуков/GROQ-запросов — установка плагина не требуется, нужна только настройка в панели управления CMS. Модули PrestaShop и Bitrix отслеживаются отдельно от основного Node.js-монорепозитория.

## Как начать работу

1. **Создайте поисковый индекс** — Панель управления → Search → Indexes → New Index
2. **Создайте токен коннектора** — Панель управления → Connectors → New Connector Token. Немедленно скопируйте открытый текст токена.
3. **Установите модуль CMS** — загрузите модуль PrestaShop или Bitrix в ваш магазин, или создайте собственную интеграцию напрямую через Connector API.
4. **Выполните рукопожатие** — модуль вызывает `POST /api/connectors/handshake` с вашим токеном. Успешный ответ подтверждает, что токен действителен, и возвращает слаг индекса.
5. **Запустите полную синхронизацию** — модуль экспортирует все товары и отправляет их пакетами на `POST /api/projects/:projectId/sync/full`. Товары помещаются в очередь `SearchIngestBuffer` и индексируются асинхронно.
6. **Встройте виджет** — скопируйте сниппет установки из Панели управления → Search → вкладка Widget и вставьте его в тему витрины.

## Принцип архитектуры

Модули CMS и браузеры никогда не обращаются к Typesense напрямую. Все данные проходят через слой API AACsearch, который валидирует, проверяет авторизацию, ограничивает запросы и ставит записи в очередь `SearchIngestBuffer`. Административный ключ Typesense никогда не покидает сервер.

```
Модуль CMS (bearer: ss_connector_*)
    │
    ▼
Connector API  ←  проверяет токен, проверяет область
    │
    ▼
SearchIngestBuffer (Prisma DB)
    │
    ▼
Фоновый воркер → Typesense
```
