> **Статус: Ранний просмотр.** Скелет модуля функционален — рукопожатие, полная синхронизация, дельта-синхронизация и встраивание виджета работают. Упаковка установки и полировка UI администратора ещё в процессе. Ожидайте шероховатостей.

Модуль AACsearch для PrestaShop (`modules/aacsearch`) интегрирует ваш каталог PrestaShop 8.x с AACsearch. Он обеспечивает:

- Экспорт товаров в AACsearch через Connector API
- Запуск дельта-синхронизации при событиях создания/обновления/удаления/изменения остатков товаров
- Встраивание хостингового виджета поиска в `<head>` вашей витрины

## Требования

- PrestaShop 8.x (совместим с PS 9 — API хуков тот же)
- PHP 7.4 или выше
- Включённое расширение cURL
- Аккаунт AACsearch с хотя бы одним созданным поисковым индексом
- Токен коннектора (`ss_connector_*`), привязанный к этому индексу

## Структура файлов модуля

```
modules/aacsearch/
  aacsearch.php                           # Главный класс модуля — установка/удаление/хуки
  config.xml                              # Дескриптор модуля (имя, версия, автор)
  index.php                               # Защита от просмотра директории
  classes/
    AacSearchClient.php                   # HTTP-клиент, оборачивающий все вызовы Connector API
    AacSearchProductExporter.php          # Нормализует товары PS → структура ProductDocument
    AacSearchSyncQueue.php                # Пакетирует товары и управляет состоянием повторов
  controllers/
    admin/
      AdminAacSearchController.php        # Страница настроек и диагностики администратора
  views/
    templates/
      admin/configure.tpl                 # Шаблон формы настроек
      hook/widget.tpl                     # Шаблон встраивания виджета
```

## Шаги установки

### 1. Загрузите модуль

Скопируйте директорию `modules/aacsearch/` в директорию `modules/` вашего PrestaShop. Затем перейдите в **Back Office → Modules → Module Manager** и установите «AACsearch».

### 2. Настройте модуль

Откройте страницу конфигурации модуля (**Modules → AACsearch → Configure**) и заполните:

| Поле                | Описание                                                                        |
| ------------------- | ------------------------------------------------------------------------------- |
| URL API AACsearch   | Базовый URL вашего экземпляра AACsearch (например, `https://app.aacsearch.com`) |
| Project ID          | ID вашей организации из панели управления AACsearch                             |
| Токен коннектора    | Токен `ss_connector_*`, созданный в Панели управления → Connectors              |
| Синхронизация вкл.  | Включить автоматическую дельта-синхронизацию при событиях товара                |
| Виджет вкл.         | Встраивать хостинговый виджет на витрину                                        |
| Локаль              | Локаль по умолчанию для результатов поиска (например, `en`, `ru`)               |
| Валюта по умолчанию | Код валюты для полей цены                                                       |
| Размер пакета       | Товаров на запрос синхронизации (по умолчанию: 200, макс.: 1000)                |
| Режим отладки       | Записывать подробные логи в `/var/logs/aacsearch.log`                           |

### 3. Проверьте соединение

Нажмите **Test Connection** на странице настроек. Это вызывает `POST /api/connectors/handshake`. Успешный ответ показывает слаг подключённого индекса и подтверждает, что токен действителен.

### 4. Запустите полную синхронизацию

Нажмите **Full Sync**, чтобы экспортировать все товары. Модуль читает товары пакетами (размер контролируется настройкой размера пакета), нормализует их с помощью `AacSearchProductExporter` и отправляет каждый пакет на `POST /api/projects/:projectId/sync/full`.

Для больших каталогов это может занять несколько минут. Прогресс и ошибки отображаются на странице диагностики.

## Структура ProductDocument

Экспортёр сопоставляет поля товаров PrestaShop со следующей структурой документа:

```json
{
	"external_id": "123",
	"title": "Blue Running Shoes",
	"description": "Lightweight running shoes for all terrains",
	"sku": "BRS-42",
	"brand": "Acme Sport",
	"categories": ["Shoes", "Running"],
	"category_ids": ["10", "22"],
	"tags": ["sale", "new-arrival"],
	"price": 99.99,
	"sale_price": 79.99,
	"currency": "EUR",
	"image_url": "https://myshop.example.com/img/brs-42.jpg",
	"product_url": "https://myshop.example.com/products/brs-42",
	"availability": "in_stock",
	"stock_quantity": 42,
	"attributes": { "color": "blue", "size": "42" },
	"locale": "en"
}
```

## Хуки

Модуль регистрирует следующие хуки PrestaShop для автоматической дельта-синхронизации:

| Хук                              | Триггер                                                              |
| -------------------------------- | -------------------------------------------------------------------- |
| `displayHeader`                  | Встраивает тег `<script>` виджета в `<head>` витрины                 |
| `actionProductUpdate`            | Срабатывает после сохранения товара — запускает дельта-синхронизацию |
| `actionObjectProductDeleteAfter` | Срабатывает после удаления товара — отправляет запрос на удаление    |
| `actionUpdateQuantity`           | Изменение остатков — запускает дельта-синхронизацию                  |

## Встраивание виджета

Когда **Виджет вкл.** отмечен, модуль выводит сниппет встраивания через хук `displayHeader`:

```html
<script
	src="https://app.aacsearch.com/api/widget/widget.js"
	data-base-url="https://app.aacsearch.com"
	data-api-key="ss_search_***"
	data-index-slug="products"
	data-container="#aac-search"
	data-theme="auto"
></script>
```

Значение `data-api-key` — это отдельный ключ `ss_search_*`, а не токен коннектора. Ключ поиска может быть только для чтения и безопасен для встраивания в браузер.

Также нужно добавить элемент-контейнер в шаблон вашей темы:

```html
<div id="aac-search"></div>
```

## Устранение неполадок

**Test Connection завершается с `invalid_or_revoked_key`**
Убедитесь, что токен коннектора скопирован правильно и не отозван в панели управления AACsearch. При необходимости создайте новый токен.

**Полная синхронизация завершена, но товары не появляются в поиске**
Синхронизация помещает товары в очередь `SearchIngestBuffer`; индексирование асинхронно. Подождите 30–60 секунд и выполните тестовый поиск. Если документы всё ещё не появляются, проверьте статус конвейера приёма данных в Панели управления → Overview.

**Виджет не отображается на витрине**
Убедитесь, что элемент контейнера `#aac-search` существует в вашей теме. Проверьте, что **Виджет вкл.** отмечен и значение `data-api-key` является действительным ключом `ss_search_*` (не токеном коннектора).
