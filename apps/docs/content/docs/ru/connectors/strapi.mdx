> **Статус: Ранний просмотр.** Скелет плагина функционален — хуки жизненного цикла, панель администратора, сопоставление полей и ручная переиндексация работают. Упаковка установки и полировка UI администратора ещё в процессе. Ожидайте шероховатостей.

Плагин AACsearch для Strapi (`@aacsearch/strapi-plugin`) интегрирует вашу CMS Strapi v5 с AACsearch. Он обеспечивает:

- Синхронизацию записей контента с AACsearch через хуки жизненного цикла (`afterCreate`, `afterUpdate`, `afterDelete`)
- Сопоставление полей — выбирайте, какие поля Strapi индексировать, и переименовывайте их
- Сопоставление типов контента — настройте несколько типов контента с разными слагами индексов
- Ручную переиндексацию — переиндексация всех записей для любого типа контента одним кликом
- Встраивание виджета — встройте хостинговый виджет поиска в ваш фронтенд

## Требования

- Strapi v5.x
- Node.js 18 или выше
- Аккаунт AACsearch с хотя бы одним созданным поисковым индексом
- Токен коннектора (`ss_connector_*`), привязанный к этому индексу

## Структура файлов плагина

```
packages/strapi-plugin/
  package.json                             # Дескриптор пакета — имя, версия, экспорты
  src/
    server/
      index.ts                             # Точка входа плагина — register/bootstrap/destroy
      lifecycles/
        index.ts                           # Регистрация хуков жизненного цикла (afterCreate/Update/Delete)
      services/
        index.ts                           # Сервис сохранения конфигурации (хранилище Strapi)
        aacsearch.ts                       # Сервис синхронизации — сопоставление документов, взаимодействие с API
        client.ts                          # HTTP-клиент — вызовы Connector API
      controllers/
        aacsearch.ts                       # Эндпоинты API администратора (get/update config, test, reindex)
    admin/
      index.ts                             # Регистрация панели администратора
      pages/
        SettingsPage.tsx                   # UI настроек — компоненты React
```

## Шаги установки

### 1. Установите плагин

```bash
npm install @aacsearch/strapi-plugin
```

Или с помощью вашего предпочтительного менеджера пакетов:

```bash
yarn add @aacsearch/strapi-plugin
pnpm add @aacsearch/strapi-plugin
```

### 2. Настройте через панель администратора

Перейдите в **Settings → AACsearch Sync** в панели администратора Strapi и заполните:

| Поле              | Описание                                                                        |
| ----------------- | ------------------------------------------------------------------------------- |
| URL API AACsearch | Базовый URL вашего экземпляра AACsearch (например, `https://api.aacsearch.com`) |
| Токен коннектора  | Токен `ss_connector_*`, созданный в Панели управления → Connectors              |
| Режим отладки     | Записывать подробные логи в логгер Strapi                                       |

После ввода данных подключения нажмите **Test Connection** для проверки.

### 3. Добавьте типы контента

Нажмите **Add Content Type** и введите UID типа контента (например, `api::product.product`). Для каждого типа контента установите **Index Slug** — он сопоставляется с именем поискового индекса в AACsearch.

Плагин автоматически генерирует слаг по умолчанию из имени типа контента (например, `api::product.product` → `product`).

### 4. Расширенная конфигурация (файл конфигурации плагина)

Для расширенного сопоставления полей создайте или отредактируйте `config/plugins.js` (или `config/plugins.ts`) в вашем проекте Strapi:

```javascript
module.exports = {
	aacsearch: {
		enabled: true,
		config: {
			baseUrl: process.env.AACSEARCH_URL,
			token: process.env.AACSEARCH_TOKEN,
			collections: {
				"api::product.product": {
					indexSlug: "products",
					fieldMapping: {
						name: "title",
						description: "body",
						price: "price",
					},
					excludeFields: ["createdBy", "updatedBy"],
				},
				"api::category.category": {
					indexSlug: "categories",
				},
			},
		},
	},
};
```

### 5. Запустите полную переиндексацию

На странице настроек администратора нажмите **Reindex** рядом с любым настроенным типом контента. Это читает все записи из Strapi, применяет ваши сопоставления полей и отправляет их в AACsearch как полную пакетную синхронизацию через `POST /api/projects/strapi/sync/full`.

Большие коллекции могут занимать несколько минут. Прогресс и ошибки отображаются в логах Strapi.

## Хуки жизненного цикла

Плагин регистрирует хуки жизненного цикла Strapi для автоматической дельта-синхронизации. Они регистрируются во время фазы `register` плагина с использованием `strapi.db.lifecycles.subscribe`:

| Хук           | Триггер                                                           |
| ------------- | ----------------------------------------------------------------- |
| `afterCreate` | Срабатывает после создания новой записи — синхронизация создания  |
| `afterUpdate` | Срабатывает после обновления записи — синхронизация обновления    |
| `afterDelete` | Срабатывает после удаления записи — отправляет запрос на удаление |

Хуки регистрируются для каждого настроенного типа контента в `src/server/lifecycles/index.ts`. Когда срабатывает событие жизненного цикла, функция `syncDocument` в `aacsearch.ts` читает конфигурацию плагина, находит настройки коллекции для типа контента, применяет сопоставления полей и вызывает Connector API через `AacSearchClient`.

## Сопоставление типов контента

Каждый тип контента в Strapi сопоставляется с поисковым индексом в AACsearch. Сопоставление определяется в `CollectionConfig`:

| Свойство        | Тип                                  | Описание                                                      |
| --------------- | ------------------------------------ | ------------------------------------------------------------- |
| `indexSlug`     | string                               | Слаг индекса AACsearch (обязательно)                          |
| `idColumn`      | string (опционально)                 | Колонка ID документа (по умолчанию: `id`)                     |
| `fieldMapping`  | Record<string, string> (опционально) | Сопоставляет имена полей Strapi с полями документов AACsearch |
| `includeFields` | string[] (опционально)               | Включать в документ только эти поля                           |
| `excludeFields` | string[] (опционально)               | Исключить эти поля из документа                               |

### Пример сопоставления полей

```json
{
	"fieldMapping": {
		"name": "title",
		"description": "body",
		"price": "price"
	}
}
```

Это сопоставляет поле Strapi `name` с `title` в документе AACsearch, `description` с `body` и `price` с `price`. Поля, не указанные в сопоставлении, исключаются, если не установлен `includeFields`.

## Эндпоинты API плагина

Плагин предоставляет следующие эндпоинты API администратора:

| Endpoint                             | Метод | Описание                              |
| ------------------------------------ | ----- | ------------------------------------- |
| `/aacsearch/get-config`              | GET   | Получить текущую конфигурацию плагина |
| `/aacsearch/update-config`           | POST  | Обновить конфигурацию плагина         |
| `/aacsearch/test-connection`         | GET   | Проверить подключение к AACsearch     |
| `/aacsearch/reindex/:contentTypeUid` | POST  | Полная переиндексация типа контента   |

## Встраивание виджета

После настройки плагина встройте хостинговый виджет AACsearch в ваш фронтенд, добавив следующий сниппет:

```html
<script
	src="https://app.aacsearch.com/api/widget/widget.js"
	data-base-url="https://app.aacsearch.com"
	data-api-key="ss_search_***"
	data-index-slug="products"
	data-container="#aac-search"
	data-theme="auto"
></script>
```

Значение `data-api-key` — это отдельный ключ `ss_search_*`, а не токен коннектора. Ключ поиска может быть только для чтения и безопасен для встраивания в браузер.

Также нужно добавить элемент-контейнер в шаблон вашего фронтенда:

```html
<div id="aac-search"></div>
```

Получите сниппет установки на вкладке **Панель управления → Search → Widget**.

## Устранение неполадок

**Test Connection завершается с `invalid_or_revoked_key`**
Убедитесь, что токен коннектора скопирован правильно и не отозван в панели управления AACsearch. При необходимости создайте новый токен.

**Переиндексация завершена, но записи не появляются в поиске**
Синхронизация помещает документы в очередь `SearchIngestBuffer`; индексирование асинхронно. Подождите 30–60 секунд и выполните тестовый поиск. Если документы всё ещё не появляются, проверьте статус конвейера приёма данных в Панели управления → Overview.

**Хуки жизненного цикла не срабатывают**
Убедитесь, что UID типа контента правильно настроен в параметрах плагина. Проверьте логи Strapi на наличие ошибок `[aacsearch]` во время регистрации жизненного цикла. Убедитесь, что плагин включён и фаза `register` завершена успешно.

**Виджет не отображается на фронтенде**
Убедитесь, что элемент контейнера `#aac-search` существует в шаблоне вашего фронтенда. Проверьте, что значение `data-api-key` является действительным ключом `ss_search_*` (не токеном коннектора).
