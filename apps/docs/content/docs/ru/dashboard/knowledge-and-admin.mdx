## Модуль знаний

> **Важное различие:** Модуль знаний — это отдельная продуктовая поверхность от витринного поиска по ключевым словам. Поиск витрины использует Typesense для быстрых запросов с устойчивостью к опечаткам по каталогам товаров. Модуль знаний предназначен для внутреннего Q&A по неструктурированным документам — PDF, DOCX, текстовым файлам — с использованием AI-ретривала (RAG и GraphRAG). Эти две поверхности независимы; они не используют общие индексы или данные.

Модуль знаний доступен по адресам:

- **Область организации:** `/[orgSlug]/knowledge` — пространства знаний для всей организации
- **Область аккаунта:** `/knowledge` — личные пространства знаний

### Пространства знаний

Пространство знаний — контейнер для источников документов и интерфейса Q&A. Вы можете создать несколько пространств (например, «Документация по товарам», «Внутренние политики», «База знаний поддержки»).

Каждое пространство принадлежит либо организации, либо пользователю (паттерн дискриминатора владельца — та же модель, что используется для поисковых индексов).

### Источники данных

Внутри пространства добавляются источники данных — в настоящее время загрузка файлов:

| Формат | Поддержка |
| ------ | --------- |
| PDF    | Поддержан |
| DOCX   | Поддержан |
| TXT    | Поддержан |

При загрузке файла создаётся задание приёма данных. Система:

1. Разбирает файл в текст
2. Разбивает текст на фрагменты с помощью настраиваемых стратегий
3. Генерирует эмбеддинги для каждого фрагмента (через OpenAI)
4. Сохраняет фрагменты в `KnowledgeChunk` с векторными эмбеддингами

### Задания приёма данных

Список заданий приёма показывает статус (running / completed / failed), имя файла, количество фрагментов и любые ошибки.

### Ask (RAG)

После завершения приёма данных можно задавать вопросы к пространству. Поток Ask:

1. Встраивает ваш запрос
2. Извлекает наиболее релевантные фрагменты через векторное сходство
3. Отправляет извлечённый контекст + вопрос в LLM
4. Возвращает ответ с цитатами источников

### GraphRAG

Функция GraphRAG строит граф знаний из принятых документов:

- `GraphNode` — концепции, сущности, темы, извлечённые из документов
- `GraphEdge` — связи между узлами

Запросы GraphRAG проходят по графу, давая более глубокие и связные ответы, чем обычный RAG. Используйте GraphRAG для вопросов, требующих рассуждений по нескольким документам или темам.

### Метрики использования

Каждое пространство отслеживает использование через процедуру `knowledge.usageMetrics`: всего запросов, потребление токенов и количество документов по источникам.

### Модели Prisma

Модуль знаний использует 7 выделенных моделей Prisma:

| Модель              | Назначение                                                   |
| ------------------- | ------------------------------------------------------------ |
| `KnowledgeSpace`    | Контейнер верхнего уровня, принадлежит орг. или пользователю |
| `DataSource`        | Запись файлового источника (имя, тип, статус)                |
| `IngestionJob`      | Отслеживание приёма данных (статус, прогресс, ошибки)        |
| `KnowledgeDocument` | Обработанный документ                                        |
| `KnowledgeChunk`    | Отдельный текстовый фрагмент для ретривала                   |
| `GraphNode`         | Узел графа знаний (концепция, сущность)                      |
| `GraphEdge`         | Связь между узлами графа                                     |

### Процедуры oRPC

Модуль знаний раскрывает 10 процедур в `packages/api/modules/knowledge/`:

| Процедура           | Описание                                    |
| ------------------- | ------------------------------------------- |
| `listSpaces`        | Список всех пространств для владельца       |
| `createSpace`       | Создать новое пространство знаний           |
| `listSources`       | Список источников данных для пространства   |
| `createSource`      | Добавить новый источник данных              |
| `listIngestionJobs` | Список заданий приёма для пространства      |
| `ingestFile`        | Загрузить и принять файл                    |
| `ask`               | Задать вопрос (ретривал RAG + ответ LLM)    |
| `graphragExplain`   | Запрос объяснения GraphRAG                  |
| `usageMetrics`      | Статистика использования токенов и запросов |

## Раздел администрирования

Раздел администрирования виден только пользователям с ролью администратора сайта. Доступен по адресу `/admin/*`.

> **Примечание:** Несколько страниц администрирования в настоящее время являются заглушками с содержимым-заполнителем — они рендерят макет страницы, но пока не имеют полной функциональности. Это отмечено по каждой странице ниже.

### Страницы администрирования

| Путь                   | Описание                                                           | Статус                                  |
| ---------------------- | ------------------------------------------------------------------ | --------------------------------------- |
| `/admin/`              | Корень панели администрирования                                    | Функционален                            |
| `/admin/organizations` | Список и управление всеми организациями                            | Функционален                            |
| `/admin/users`         | Список и управление всеми пользователями, имперсонация             | Функционален                            |
| `/admin/config`        | Системная конфигурация                                             | Заглушка — ожидает                      |
| `/admin/security`      | Настройки безопасности и управление сессиями                       | Заглушка — ожидает                      |
| `/admin/integrations`  | Настройки интеграций с третьими сторонами                          | Заглушка — ожидает                      |
| `/admin/audit`         | Просмотр журнала аудита                                            | Заглушка — ожидает                      |
| `/admin/wallet`        | Администрирование AI-кошелька (баланс, транзакции на пользователя) | Заглушка — в активной разработке (v0.6) |
| `/admin/jobs`          | Управление фоновыми заданиями                                      | Заглушка — ожидает                      |
| `/admin/notifications` | Управление системными уведомлениями                                | Заглушка — ожидает                      |

### AI-кошелёк и кредиты (v0.6 — в активной разработке)

Система кошелька (`@repo/billing-wallet`) управляет балансами AI-кредитов с использованием леджера BigInt в копейках. Страница администрирования кошелька по адресу `/admin/wallet` и пользовательская страница AI-кредитов по адресу `/settings/billing/ai-credits` активно разрабатываются как часть релиза тарификации v0.6. Драйвер платежей Tochka (для рынка РФ) подключён; Stripe является параллельным вариантом.

Функциональность кошелька намеренно полуизолирована — если ваша задача не связана с биллингом или AI-кредитами, не расширяйте её без явного подтверждения области.

## Настройки на уровне аккаунта

Личные настройки находятся вне контекста организации по адресу `/settings/`.

| Путь                           | Содержание                                            |
| ------------------------------ | ----------------------------------------------------- |
| `/settings/general`            | Отображаемое имя, email, фото профиля                 |
| `/settings/security`           | Смена пароля, 2FA (TOTP), passkeys, OAuth-подключения |
| `/settings/notifications`      | Настройки уведомлений по типу                         |
| `/settings/billing`            | Личный биллинг, активный план, способ оплаты          |
| `/settings/billing/ai-credits` | Пополнение AI-кошелька и история транзакций (v0.6)    |

Настройки безопасности поддерживают: вход по паролю, magic link, двухфакторную аутентификацию (TOTP через приложение-аутентификатор), passkey (WebAuthn) и OAuth (Google, GitHub).
