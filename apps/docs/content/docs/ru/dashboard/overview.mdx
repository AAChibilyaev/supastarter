Панель AACsearch — это защищённое SaaS-приложение в `apps/saas`. Оно организовано в две области: **уровень организации** (ваше рабочее пространство для поиска) и **уровень аккаунта** (ваш личный аккаунт и инструменты администрирования).

## Области панели управления

### Область организации (`/[orgSlug]/...`)

Всё, что связано с поиском для конкретной организации, находится здесь. Организация соответствует арендатору в AACsearch — все индексы, API-ключи, аналитика и биллинг привязаны к ней.

| Путь                          | Область                                                                      |
| ----------------------------- | ---------------------------------------------------------------------------- |
| `/[orgSlug]/overview`         | Панель KPI, топ запросов, лента активности, предупреждения о квоте           |
| `/[orgSlug]/getting-started`  | 6-шаговый чек-лист онбординга                                                |
| `/[orgSlug]/search`           | Рабочее пространство поиска (вкладки: Indexes, Playground, API Keys, Widget) |
| `/[orgSlug]/api-keys`         | API-ключи для всех индексов                                                  |
| `/[orgSlug]/import-jobs`      | История заданий импорта                                                      |
| `/[orgSlug]/preview`          | Превью / playground для поиска                                               |
| `/[orgSlug]/analytics`        | Аналитика поиска — CTR, топ запросов, сессии                                 |
| `/[orgSlug]/relevance`        | Синонимы, кураторство, стоп-слова, пресеты                                   |
| `/[orgSlug]/connectors`       | Статус модулей CMS, токены коннектора, история заданий синхронизации         |
| `/[orgSlug]/knowledge`        | Пространства знаний (RAG/GraphRAG — отдельная продуктовая поверхность)       |
| `/[orgSlug]/settings/general` | Настройки организации                                                        |
| `/[orgSlug]/settings/members` | Управление участниками, приглашения                                          |
| `/[orgSlug]/settings/billing` | Биллинг и план организации                                                   |

### Область аккаунта (`/...`)

Личные настройки и инструменты администрирования находятся вне контекста организации.

| Путь                           | Область                                                           |
| ------------------------------ | ----------------------------------------------------------------- |
| `/settings/general`            | Профиль аккаунта                                                  |
| `/settings/security`           | Пароль, 2FA (TOTP, passkeys), OAuth-подключения                   |
| `/settings/notifications`      | Настройки уведомлений                                             |
| `/settings/billing`            | Личный биллинг                                                    |
| `/settings/billing/ai-credits` | AI-кошелёк (в активной разработке — v0.6)                         |
| `/knowledge`                   | Личные пространства знаний                                        |
| `/chatbot`                     | AI-чатбот                                                         |
| `/admin/*`                     | Инструменты администрирования сайта (только для admin — см. ниже) |

## Страница Overview

Страница обзора организации (`/[orgSlug]/overview`) — основная страница после выбора организации. Она показывает:

### Плитки KPI

Четыре сводных метрики за выбранный период (24 ч / 7 д / 30 д):

| Плитка                      | Источник                                             |
| --------------------------- | ---------------------------------------------------- |
| Всего поисков               | Потребление поисковых единиц из прав доступа к плану |
| Проиндексировано документов | Количество документов из прав доступа к плану        |
| Использование квоты         | `searches.percentUsed` из `entitlements.plan`        |
| Неудавшихся синхронизаций   | Derived из статуса конвейера приёма данных           |

Предупреждающий баннер появляется, когда квота достигает 80%, и баннер ошибки — когда она превышена.

### Поиски во времени

Искровый график, показывающий объём поиска за выбранный период. Данные берутся из строк `SearchUsageEvent` через процедуру oRPC `search.usage`.

### Топ запросов

Ранжированный список 10 наиболее частых поисковых запросов за период. Питается процедурой `search.topQueries`, группирующей строки `SearchUsageEvent` по тексту запроса.

### Лента активности

Упорядоченный по времени список недавних событий, производных от временных меток создания индексов и количества API-ключей. Выделенная процедура `recentActivity` запланирована для v1.x; текущая лента производится из существующих данных запросов.

### Статус коннектора

Показывает, вызывал ли какой-либо коннектор эндпоинт heartbeat недавно. Статус определяется по `lastUsedAt` токенов коннектора:

- **Online** — последний heartbeat в течение 5 минут
- **Unknown** — последний heartbeat в течение 30 минут
- **Offline** — нет недавнего heartbeat

### Карточка плана

Показывает активное название плана и статус (active / canceled / past_due). Ссылается на страницу биллинга.

## Выбор периода

Страница обзора поддерживает выбор периода (24 ч / 7 д / 30 д), хранимый в URL как `?period=7d`. Это управляет всеми агрегациями KPI на странице.

## Структура навигации

Боковая панель на уровне организации содержит 9 пунктов:

1. Overview
2. Getting Started
3. Search
4. Analytics
5. Relevance
6. Connectors
7. Knowledge
8. Settings
9. (Billing — вложено в Settings)
