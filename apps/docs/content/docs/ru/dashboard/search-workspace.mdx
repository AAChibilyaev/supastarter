Рабочее пространство Search по адресу `/[orgSlug]/search` — главная панель управления для ваших поисковых индексов. Оно организовано во вкладки:

- **Indexes** — управление поисковыми индексами
- **Playground** — интерактивное тестирование запросов
- **API Keys** — управление поисковыми API-ключами по индексам
- **Widget** — получение сниппета установки хостингового виджета

## Вкладка Indexes

Вкладка Indexes перечисляет все поисковые индексы организации. Для каждого индекса можно:

- Просмотреть количество документов и временную метку последнего обновления
- Открыть действия строки: переиндексация, просмотр заданий импорта, управление схемой, синонимами, кураторством
- Создать новый индекс (только для администраторов организации)

Карточка сводки использования вверху показывает потребление относительно лимитов плана (использовано поисков, проиндексировано документов, процент квоты).

### Создание индекса

Нажмите **New Index** (доступно администраторам организации). Вы выбираете отображаемое имя и слаг. Слаг становится частью имени коллекции Typesense и не может быть изменён после создания.

Слаги индексов используются в:

- Атрибуте виджета `data-index-slug`
- Сегменте пути `:indexSlug` Connector API
- Привязках API-ключей

### Переиндексация

Запуск переиндексации из действий строки создаёт новую версионированную коллекцию Typesense (`{orgShortId}_{slug}_v{n+1}`), проверяет её и атомарно переключает псевдоним. Предыдущая версия остаётся активной до завершения переключения. Никакого простоя.

## Вкладка Playground

Playground позволяет выполнять тестовые запросы к любому индексу без написания кода. Вы можете:

- Вводить поисковые запросы
- Применять фильтры
- Пробовать разные варианты сортировки
- Просматривать сырые результаты Typesense, включая количества фасетов и подсветку

Playground использует вашу сессию панели управления — API-ключ не требуется.

## Вкладка API Keys

Вкладка API Keys показывает все ключи для выбранного индекса. Выберите индекс из выпадающего списка, чтобы увидеть его ключи.

### Области ключей

| Область           | Назначение                                                           |
| ----------------- | -------------------------------------------------------------------- |
| `search`          | Поисковые запросы только для чтения. Безопасно встраивать в браузер. |
| `ingest`          | Доступ на запись для отправки документов через REST API              |
| `admin`           | Полный доступ — создание/удаление коллекций, управление схемой       |
| `connector_write` | Используется модулями CMS (см. токены коннектора ниже)               |

### Создание ключа

Нажмите **Create Key** и выберите имя и область. Открытый текст ключа показывается один раз при создании. Скопируйте его немедленно — он не может быть получен снова.

Ключи хранятся как SHA-256 хэши (формат prefix:hash). В открытом виде хранятся только префикс и метаданные.

### Отзыв ключа

Нажмите **Revoke** на любом активном ключе. Отзыв действует немедленно — ключ отклоняется при следующем использовании. Отозванные ключи сохраняются в списке для целей аудита.

## Вкладка Widget

Вкладка Widget генерирует сниппет установки хостингового виджета. Выберите индекс из выпадающего списка, затем:

1. Введите ваш API-ключ `ss_search_*` (или используйте уже созданный ключ)
2. Скопируйте сгенерированный сниппет

Сниппет выглядит так:

```html
<script
	src="https://app.aacsearch.com/api/widget/widget.js"
	data-base-url="https://app.aacsearch.com"
	data-api-key="ss_search_***"
	data-index-slug="products"
	data-container="#aac-search"
	data-theme="auto"
></script>
```

Вставьте его в `<head>` вашей витрины и добавьте `<div id="aac-search"></div>` туда, где должен появляться поиск.

Полные параметры конфигурации смотрите в разделе [Обзор виджета](/docs/connectors/widget-overview).

## Import Jobs

Страница `/[orgSlug]/import-jobs` показывает историю операций массового импорта.

### Массовый импорт

Вы можете загрузить JSONL-файл из вкладки Indexes → действия строки → Import Documents. Каждая строка файла — это документ товара в той же форме, что и данные синхронизации Connector API:

```jsonl
{"external_id":"1","title":"Product A","price":9.99}
{"external_id":"2","title":"Product B","price":19.99}
```

Задания импорта отслеживаются в базе данных. Каждое задание показывает:

- Статус (running / completed / failed)
- Всего обработанных элементов
- Количество неудач
- Временные метки начала и завершения

## Чек-лист Getting Started

Страница `/[orgSlug]/getting-started` показывает 6-шаговый чек-лист, который автоматически определяет своё состояние на основе ваших существующих данных — никакого ручного ввода не требуется.

| Шаг                               | Условие                                                                   |
| --------------------------------- | ------------------------------------------------------------------------- |
| 1. Создать поисковый индекс       | Количество `SearchIndex` > 0 для организации                              |
| 2. Сгенерировать токен коннектора | Существует `SearchApiKey` с областью `connector_write` и не отозван       |
| 3. Синхронизировать данные        | Существует `SearchUsageEvent` с типом ingest                              |
| 4. Выполнить поиск                | Существует `SearchUsageEvent` с типом search                              |
| 5. Сгенерировать API-ключ         | Существует `SearchApiKey` с областью `search` и не отозван                |
| 6. Встроить виджет поиска         | Всегда начинается неотмеченным — самостоятельная аттестация пользователем |

Шаги 1–5 проверяются автоматически. Шаг 6 отражает встраивание виджета, которое не может быть обнаружено на стороне сервера.

## Токены коннектора

Токены коннектора (`ss_connector_*`) управляются со страницы Connectors по адресу `/[orgSlug]/connectors`, а не из рабочего пространства Search. Оттуда вы можете:

- Просматривать все активные токены коннектора с временем последнего heartbeat
- Создавать новый токен коннектора для конкретного индекса
- Отзывать токен

Полный рабочий процесс коннектора смотрите в разделе [Обзор коннекторов](/docs/connectors/overview).

## Настройки релевантности

Страница `/[orgSlug]/relevance` содержит вкладки для настройки релевантности по индексам:

- **Synonyms** — добавление наборов синонимов (например, «кроссовки» = «беговые кеды» = «спортивная обувь»)
- **Curations** — закрепление или скрытие конкретных документов для определённых запросов
- **Stopwords** — определение слов для игнорирования при поиске (запланировано для v1.x)
- **Presets** — сохранение именованных наборов параметров поиска для быстрого повторного использования (запланировано для v1.x)

## Страница Analytics

Страница `/[orgSlug]/analytics` предоставляет детальную аналитику поиска для организации. Она показывает общее количество поисков, уникальные сессии, процент нулевых результатов, среднюю задержку и кликабельность (CTR). Данные берутся из строк `SearchUsageEvent` через процедуру oRPC `search.analytics`.
