title: API-ключи

---

AACsearch использует три типа API-ключей, каждый с уникальным префиксом и областью разрешений.
Все ключи хранятся в виде bcrypt-хэшей — открытый текст отображается **один раз** при создании и больше не может быть получен.

## Типы ключей

| Тип             | Префикс          | Области             | Используется                                      |
| --------------- | ---------------- | ------------------- | ------------------------------------------------- |
| Ключ поиска     | `ss_search_*`    | `search`            | Браузерный SDK, виджет, прямые вызовы API         |
| Ключ коннектора | `ss_connector_*` | `connector_write`   | Модули CMS (PrestaShop, Bitrix)                   |
| Ограниченный токен | `ss_scoped_*` | Сужение от ключа поиска | На пользователя, с ограничением по фильтру  |

## Генерация ключа поиска

### Через панель управления

1. Перейдите в **Search** → **API Keys**
2. Нажмите **Create API key**
3. Выберите тип ключа: **Search**
4. Настройте разрешённые источники (оставьте пустым при разработке)
5. Установите ограничение запросов (по умолчанию: 60 зап./мин.)
6. При желании задайте срок истечения
7. Нажмите **Create** — скопируйте ключ немедленно, он больше не будет показан

### Через oRPC

```typescript
const key = await orpc.search.createApiKey.call({
	organizationId: "org_...",
	indexSlug: "products",
	scopes: ["search"],
	allowedOrigins: ["https://yourstore.com"],
	rateLimitPerMinute: 60,
	name: "Storefront search key",
});

console.log(key.plaintext); // ss_search_abc123... — сохраните сейчас!
```

## Генерация ключа коннектора

Ключи коннектора используют область `connector_write` и префикс `ss_connector_*`.
Они используются модулями CMS для отправки документов на эндпоинт приёма данных.

```typescript
const key = await orpc.search.createConnectorToken.call({
	organizationId: "org_...",
	indexSlug: "products",
	name: "PrestaShop connector",
});

console.log(key.plaintext); // ss_connector_abc123...
```

В панели управления ключи коннектора отображаются в **Connectors** → **Connector Tokens**.

## Использование ключа поиска

### HTTP-заголовок

```bash
curl -X POST https://your-app.com/api/search \
  -H "Authorization: Bearer ss_search_your_key" \
  -H "Content-Type: application/json" \
  -d '{ "indexSlug": "products", "q": "headphones" }'
```

### Браузерный SDK

```typescript
import { AacSearchClient } from "@repo/search-client";

const client = new AacSearchClient({
	baseUrl: "https://your-app.com",
	apiKey: "ss_search_your_key",
	indexSlug: "products",
});

const results = await client.search({ q: "headphones" });
```

Полную документацию SDK смотрите в разделе [Браузерный SDK](/docs/getting-started/browser-sdk).

## Ограниченные токены

Ограниченные токены — это краткосрочные, подписанные HMAC-токены, сужающие разрешения ключа поиска.
Они генерируются на стороне сервера и передаются браузеру — базовый ключ поиска остаётся на сервере.

**Типичный сценарий:** Ограничить результаты поиска организацией или ценовым сегментом текущего пользователя.

```typescript
// Сторона сервера: генерация ограниченного токена
const scopedToken = await orpc.search.createScopedToken.call({
	organizationId: "org_...",
	indexSlug: "products",
	scopedFilter: "availability:=in_stock && price:<100",
	expiresInSeconds: 3600,
});

// Передайте браузеру — используйте как обычный ключ поиска
// Фильтр объединяется через AND с любыми фильтрами вызывающей стороны
```

Ограниченные токены не хранятся в БД (stateless). Они истекают по параметру `expiresInSeconds`.
Подробности смотрите в разделе [Ограниченные поисковые токены](/docs/search-api/scoped-search-tokens).

## Ограничение по источнику (Origin)

Каждый ключ поиска имеет список `allowedOrigins[]`. Запросы с незарегистрированных источников получают ответ 403.

- **Разработка:** оставьте `allowedOrigins` пустым, чтобы разрешить все источники
- **Production:** явно укажите домены вашего магазина

```typescript
// Пример: ограничение конкретными доменами
allowedOrigins: ["https://mystore.com", "https://www.mystore.com"];
```

**Ограничение бесплатного плана:** На бесплатном плане в `allowedOrigins` допустимы только поддомены `*.aacsearch.com`.
Для поддержки пользовательских доменов перейдите на план Pro.

## Ограничение запросов

Ограничение по умолчанию: **60 запросов в минуту** на ключ. При превышении возвращается:

```json
HTTP/1.1 429 Too Many Requests
Retry-After: 15

{ "error": "rate_limit_exceeded" }
```

Настройте лимит для каждого ключа при создании или обновите его в панели управления.

## Отзыв ключа

```typescript
await orpc.search.revokeApiKey.call({
	organizationId: "org_...",
	keyId: "key_...",
});
```

Отзыв ключа действует немедленно. Ключ будет отклонён при следующем запросе. Хэшированное значение сохраняется в базе данных для целей аудита.

## Чек-лист безопасности ключей

- [ ] Никогда не фиксируйте ключи в системе контроля версий
- [ ] Никогда не логируйте ключи (AACsearch никогда не логирует их на стороне сервера)
- [ ] Используйте ключи `ss_search_*` в браузере (только чтение, ограничено источником)
- [ ] Используйте ключи `ss_connector_*` только в модуле CMS (на стороне сервера)
- [ ] Периодически ротируйте ключи через процедуру отзыва + повторного создания
- [ ] Устанавливайте `allowedOrigins` для production-ключей
- [ ] Устанавливайте `expiresAt` для временного доступа
