В этом разделе описан путь от нуля до рабочего поискового эндпоинта в AACsearch. Независимо от того, являетесь ли вы разработчиком, настраивающим монорепозиторий локально, или продавцом, подключающим магазин PrestaShop, — начните именно здесь.

## Путь к рабочему поиску

```
1. Создать аккаунт + организацию
2. Создать поисковый индекс
3. Сгенерировать API-ключи
4. Загрузить первые документы
5. Выполнить поисковый запрос
6. (Опционально) Установить виджет на витрину магазина
7. (Опционально) Подключить модуль CMS для автоматической синхронизации
```

## Роли и их потребности

### Разработчик / интегратор

Вы хотите настроить локальный монорепозиторий, разобраться с API и подключить собственную интеграцию.

Начните с:

1. [Локальная разработка](/docs/getting-started/local-development) — настройка среды
2. [Создание первого индекса](/docs/getting-started/create-first-index) — понимание модели индекс + псевдоним
3. [API-ключи](/docs/getting-started/api-keys) — разобраться с типами ключей перед обращением к API
4. [Браузерный SDK](/docs/getting-started/browser-sdk) — поиск из браузера или Node.js-клиента
5. [Приём данных и переиндексация](/docs/getting-started/ingest-and-reindex) — загрузка и обновление документов

### Продавец / оператор магазина

Вы хотите подключить существующий магазин PrestaShop или Bitrix и запустить поиск на витрине.

Начните с:

1. Зарегистрируйтесь в панели управления AACsearch
2. Создайте организацию и поисковый индекс
3. [Подключение коннектора](/docs/getting-started/connector-onboarding) — сгенерируйте токен коннектора и подключите CMS
4. [Установка виджета](/docs/getting-started/widget-installation) — добавьте виджет поиска на витрину

## Предварительные требования

| Требование         | Примечания                                             |
| ------------------ | ------------------------------------------------------ |
| Node.js 20+        | Для локальной разработки                               |
| pnpm 10            | Менеджер пакетов — установка: `npm i -g pnpm`          |
| Docker             | Для локальных Postgres + MinIO (объектное хранилище)   |
| Экземпляр Typesense | Docker-образ или управляемое облако                   |
| Аккаунт AACsearch  | Зарегистрируйтесь в панели управления                  |

## Определение готовности MVP

Полный путь end-to-end для AACsearch MVP:

```
Регистрация → создание организации → создание проекта →
генерация токена коннектора → установка модуля PrestaShop или Bitrix →
подключение модуля → запуск полной синхронизации → товары проиндексированы →
тест поиска в панели управления → включение виджета → поиск на витрине работает →
базовая аналитика появляется в панели управления
```

Текущее состояние каждого шага смотрите в разделе [Статус и дорожная карта](/docs/overview/status-roadmap).
