title: Установка виджета

---

Виджет AACsearch — это самодостаточный Vanilla JS-компонент, который добавляет полноценный поиск
на любую витрину с помощью одного тега `<script>`. Он использует Shadow DOM для изоляции CSS
и весит около 14 КБ в минифицированном виде (сборки IIFE + ESM).

## Как работает виджет

```
Загружается тег <script data-*>
    │
    ▼
Инициализация виджета (читает атрибуты data-*)
    │
    ▼
Создаётся контейнер Shadow DOM (изоляция CSS)
    │
    ▼
GET /api/rpc/search.widgetConfig  ←  загружает конфигурацию времени выполнения (макет, фильтры, параметры сортировки)
    │
    ▼
Пользователь вводит текст → POST /api/search  ←  ключ только для поиска в заголовке Authorization
    │
    ▼
Результаты рендерятся в Shadow DOM
    │
    ▼
Отслеживаются события → POST /api/events/track (асинхронно, keepalive)
```

Виджет использует только ключ `ss_search_*`, встроенный в `data-api-key`. Административный ключ Typesense никогда не попадает в браузер.

## Базовая установка

Добавьте тег script в `<head>` вашей витрины или перед `</body>`:

```html
<script
	src="https://your-app.com/api/widget/widget.js"
	data-base-url="https://your-app.com"
	data-api-key="ss_search_your_search_key"
	data-index-slug="products"
	data-container="#search-container"
	data-theme="auto"
></script>
```

Создайте элемент-контейнер, в который будет монтироваться виджет:

```html
<div id="search-container"></div>
```

## Атрибуты тега script

| Атрибут           | Обязателен | По умолчанию | Описание                                          |
| ----------------- | ---------- | ------------ | ------------------------------------------------- |
| `data-base-url`   | Да         | —            | Базовый URL вашего API AACsearch                  |
| `data-api-key`    | Да         | —            | Ключ `ss_search_*` для данного индекса            |
| `data-index-slug` | Да         | —            | Слаг индекса (например, `products`)               |
| `data-container`  | Да         | —            | CSS-селектор элемента монтирования                |
| `data-theme`      | Нет        | `auto`       | `light`, `dark` или `auto`                        |
| `data-locale`     | Нет        | `en`         | Локаль UI виджета (`en`, `de`, `es`, `fr`, `ru`)  |
| `data-layout`     | Нет        | `inline`     | `inline` или `modal`                              |

## Режимы макета

### Встроенный (Inline)

Виджет рендерит строку поиска и панель результатов непосредственно в контейнере. Контейнер должен иметь определённую высоту, если результаты занимают много места.

```html
<div id="search-container" style="min-height: 400px;"></div>
```

### Модальный (Modal)

Виджет рендерит только кнопку запуска поиска в контейнере. По нажатию открывается полноэкранный оверлей поиска. Подходит для поиска в шапке сайта.

```html
<script ... data-layout="modal" data-container="#search-trigger"></script>
<button id="search-trigger">Search</button>
```

## Кастомизация темы

Виджет поддерживает три режима темы:

- `auto` — следует предпочтению тёмного режима ОС/браузера
- `light` — всегда светлая
- `dark` — всегда тёмная

CSS-переменные для тонкой настройки темы (задаются на странице-хосте, проникают через границу Shadow DOM
только если виджет открывает CSS parts):

```css
/* Отражаются в теневом корне виджета */
#search-container {
	--aac-accent: #6366f1;
	--aac-border-radius: 8px;
}
```

## Установка через модуль CMS

При использовании модуля PrestaShop или Bitrix тег script виджета **автоматически встраивается** модулем в `<head>` витрины через систему хуков CMS. Добавлять его вручную не нужно.

Модуль считывает конфигурацию виджета (URL API, ключ, слаг индекса) со страницы настроек модуля.
Подробности смотрите в разделах [PrestaShop](/docs/connectors/prestashop) и [Bitrix](/docs/connectors/bitrix).

## Получение сниппета виджета из панели управления

Панель управления генерирует предзаполненный сниппет в **Search** → **Widget**:

1. Перейдите к вашему поисковому индексу
2. Нажмите вкладку **Widget**
3. Скопируйте сгенерированный тег `<script>` с предзаполненными ключом и слагом индекса

Компонент `WidgetPanel` в панели управления (`apps/saas/modules/search/components/WidgetPanel.tsx`)
также отображает токен коннектора для конфигурации модуля CMS.

## Политика безопасности содержимого (CSP)

Если на вашей витрине используется заголовок CSP, добавьте источник скрипта AACsearch в политику:

```
script-src 'self' https://your-app.com;
connect-src 'self' https://your-app.com;
```

Shadow DOM не требует `style-src unsafe-inline` — стили изолированы в теневом корне.

## Поддержка браузеров

Виджет ориентирован на современные браузеры (Chrome 88+, Firefox 85+, Safari 14+, Edge 88+).
Требуется Shadow DOM v1. Internet Explorer не поддерживается.

## Отслеживание аналитики

Виджет автоматически отслеживает следующие события (отправляются на `POST /api/events/track`):

- `widget_open` — при открытии виджета
- `search_query` — при выполнении поиска
- `zero_results` — когда запрос не возвращает результатов
- `result_click` — при нажатии на результат
- `filter_used` — при применении фильтра-фасета

Полную схему событий смотрите в разделе [Аналитические события виджета](/docs/connectors/widget-analytics-events).
