AACsearch построен на **монорепозитории supastarter Next.js** с использованием Turborepo и pnpm workspaces.
Кодовая база разделена на четыре приложения и шестнадцать пакетов, все обслуживаются через единый HTTP-процесс Hono,
который монтирует oRPC, публичные поисковые эндпоинты, бандл виджета, коннекторы и аутентификацию.

## Структура монорепозитория

```
apps/
  saas/           # Защищённая панель SaaS — индексы, ключи, аналитика, коннекторы, биллинг
  marketing/      # Публичный маркетинговый сайт — лендинг, цены, блог, changelog, правовые страницы
  docs/           # Публичный сайт документации (этот сайт — v0.7)
  mail-preview/   # Только предпросмотр шаблонов электронной почты

packages/
  api/            # Hono + oRPC — 11 модулей oRPC + публичные маршруты Hono
  auth/           # Better Auth — вход, орг., passkeys, 2FA, magic links, OAuth
  database/       # Prisma (активная ORM) + Drizzle (только для справки) — 33 модели
  search/         # Клиент Typesense, коллекции, буфер, переиндексация, ключи, приём данных
  search-client/  # Браузерный SDK — только поиск, без административных ключей
  widget/         # Хостинговый виджет витрины — Vanilla JS, Shadow DOM, 14 КБ IIFE+ESM
  i18n/           # 5 локалей × 4 области (en/de/es/fr/ru × marketing/saas/mail/shared)
  payments/       # Stripe / LemonSqueezy / Polar / Creem / DodoPayments / Tochka
  billing-wallet/ # Леджер BigInt в копейках, reserve→commit/release
  ai/             # Vercel AI SDK + конфигурации провайдеров
  ai-core/        # Нижнеуровневые примитивы AI-оркестрации
  notifications/  # createNotification, list, mark-read, preferences, catalog
  mail/           # Шаблоны React Email + драйверы провайдеров
  storage/        # S3-совместимое хранилище (MinIO для локальной разработки)
  ui/             # 27 UI-примитивов на основе shadcn
  logs/           # Логгер на основе pino
  utils/          # Общие утилиты

modules/          # НЕ Node-воркспейс — PHP-модули CMS
  prestashop/aacsearch/   # PrestaShop 8.x (скелет, отдельный трек)
  bitrix/aac.search/      # 1C-Bitrix self-hosted (скелет, отдельный трек)
```

## HTTP-сервер

Весь трафик поступает через **единственное приложение Hono** в `packages/api/index.ts`, которое монтирует маршруты в следующем порядке:

| Монтирование             | Назначение                                     |
| ------------------------ | ---------------------------------------------- |
| `/api`                   | Публичный обработчик поиска (CORS без ограничений) |
| `/api`                   | Аналитические события виджета                  |
| `/api/widget/widget.js`  | Хостинговый JS-бандл виджета                   |
| `/api`                   | Connector API (модули CMS)                     |
| `/api`                   | Обработчик аналитики                           |
| `/api`                   | SCIM 2.0 (провизия идентификационных данных)   |
| `/api/v1`                | REST API v1 (15 эндпоинтов, OpenAPI 3.1)       |
| `/api/auth/**`           | Обработчик Better Auth                         |
| `/api/webhooks/payments` | Вебхуки платёжных провайдеров                  |
| `/api/rpc/**`            | Обработчик oRPC (11 модулей)                   |

## Модули oRPC

Роутер oRPC в `packages/api/orpc/router.ts` монтирует 11 модулей:

`admin`, `organizations`, `users`, `payments`, `ai`, `notifications`, `search`, `knowledge`, `billingWallet`, `entitlements`

Модуль `search` содержит 26 процедур, охватывающих индексы, API-ключи, ограниченные токены, релевантность, аналитику, коннекторы и задания импорта.

## Принцип многоуровневого бэкенда

Внешние модули CMS и браузеры **никогда** не обращаются к Typesense напрямую. Единственный держатель административных учётных данных — слой API AACsearch.

```
Модуль PrestaShop / Bitrix
    │  bearer(ss_connector_* token)
    ▼
Connector API AACsearch  ←  валидация, проверка авторизации, ограничение запросов
    │
    ▼
SearchIngestBuffer (Prisma)  ←  enqueueManySearchIngest()
    │
    ▼
Фоновый воркер  ←  дренирует буфер пакетами
    │
    ▼
Typesense  (только на стороне сервера, административный ключ никогда не покидает сервер)
```

Раскрытие административного ключа Typesense в модуле CMS или теге `<script>` открыло бы возможность произвольных манипуляций с индексами всех арендаторов. Это **Hard Invariant #2** в кодовой базе, не подлежащий обсуждению.

## Путь записи (приём данных)

```
Запрос → public-handler.ts (проверка авторизации + квоты)
  → enqueueManySearchIngest() (вставка в SearchIngestBuffer)
  → фоновый воркер подхватывает необработанные строки
  → bulkUpsert в Typesense
  → markIngestRowsSuccess / markIngestRowsFailure (экспоненциальная задержка при сбоях)
```

Подход через буфер БД обеспечивает надёжность (строки переживают перезапуск сервера), обработку частичных сбоев
(только неудавшиеся строки повторяются) и пакетирование (воркер группирует строки по коллекции перед отправкой).

## Путь чтения (поиск)

```
Запрос → public-handler.ts (авторизация + ограничение запросов + проверка квоты)
  → searchDocuments / multiSearchDocuments (через packages/search/lib/search.ts)
  → ответ Typesense
  → возврат очищенного JSON (сырые ошибки Typesense никогда не пересылаются)
  → recordSearchUsage() вызывается асинхронно (не блокирует)
```

## Именование и версионирование коллекций

Каждый поисковый индекс использует версионированные имена коллекций для обеспечения переиндексации без простоя:

```
Имя коллекции:  {orgShortId}_{slug}_v{version}
Имя псевдонима: {orgShortId}_{slug}
```

Все поисковые запросы направляются к псевдониму. При переиндексации создаётся новая версия, проверяется, затем псевдоним атомарно переключается. Предыдущая версия сохраняется до следующей успешной переиндексации.

## Выбор технологий

| Слой                  | Технология                                       |
| --------------------- | ------------------------------------------------ |
| Фреймворк             | Next.js 16 (App Router, RSC)                     |
| API                   | Hono + oRPC 1.13                                 |
| Аутентификация        | Better Auth 1.5                                  |
| ORM                   | Prisma 7 (активная) / Drizzle (только для справки) |
| Поисковый движок      | Typesense 3                                      |
| Валидация             | Zod 4                                            |
| Загрузка данных (клиент) | TanStack Query 5                              |
| Формы                 | react-hook-form 7                                |
| UI                    | Tailwind v4 + shadcn примитивы                   |
| Линтинг               | Oxlint (не ESLint)                               |
| Форматирование        | Oxfmt (не Prettier)                              |
| Менеджер пакетов      | pnpm 10 с каталогом воркспейса                   |
| Оркестрация сборки    | Turborepo 2                                      |
