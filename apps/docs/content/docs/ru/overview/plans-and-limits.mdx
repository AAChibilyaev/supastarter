AACsearch применяет ограничения по планам через систему прав доступа на базе `@repo/payments/lib/entitlements`.
Отдельной таблицы счётчика использования пока нет — квота проверяется по конфигурации в момент запроса и валидируется
по сырым строкам `SearchUsageEvent` для агрегации по периодам.

**Примечание:** Приведённые ниже цифры — это черновая позиция, а не публично подтверждённые тарифы. Инфраструктура применения поставлена; точные границы планов могут измениться до запуска.

## Планы (черновые)

| План           | Поисковых единиц/мес. | Макс. индексов  | Список источников                | Поддержка          |
| -------------- | --------------------- | --------------- | -------------------------------- | ------------------ |
| **Free**       | 10 000                | 1               | Только поддомены `aacsearch.com` | Сообщество         |
| **Starter**    | 100 000               | 3               | Пользовательские источники       | Email              |
| **Pro**        | 1 000 000             | 10              | Пользовательские источники       | Приоритетный email |
| **Business**   | 5 000 000             | 50              | Пользовательские источники       | Выделенная         |
| **Enterprise** | Без ограничений       | Без ограничений | Пользовательские источники       | SLA 99,95%         |

Одна **поисковая единица** = 1 поисковый запрос ИЛИ 1 проиндексированный документ.

## Фичер-гейты

Доступность функций сверх квоты контролируется `packages/api/modules/entitlements/middleware/feature-gate.ts`.
Процедуры oRPC используют middleware:

```typescript
// Ограничить процедуру флагом функции
.use(featureGate("synonyms"))

// Ограничить операцию записи
.use(writeGate("synonyms"))
```

Матрица функций сопоставляет ID планов с включёнными функциями в `@repo/payments/lib/entitlements`:

```typescript
const FEATURE_MATRIX: Record<PlanId, Features> = {
  free:       { synonyms: false, curations: false, originRestriction: false, ... },
  starter:    { synonyms: true,  curations: true,  originRestriction: false, ... },
  pro:        { synonyms: true,  curations: true,  originRestriction: true,  ... },
  enterprise: { synonyms: true,  curations: true,  originRestriction: true,  ... },
};
```

## Поток применения квоты

Каждый запрос поиска и приёма данных проходит через:

```
Запрос
  → Auth gate (действительный ключ + орг.)
  → Feature gate (план разрешает операцию)
  → Quota gate (у организации остались единицы)
  → Rate gate (лимит запросов по ключу не превышен)
  → Typesense
```

Middleware `quotaCheck` в `packages/api/modules/entitlements/middleware/quota-check.ts`
вызывает `checkQuota(orgId, "search")` и возвращает:

- `allowed: true` — продолжить
- `allowed: false`, `reason: "search_quota_exceeded"` — ответ 429

Жёсткие ограничения немедленно блокируют весь трафик. Мягкие ограничения (настраиваемый льготный период) могут разрешить ограниченное чтение
на льготный период после отмены (по умолчанию: 7 дней для чтения, 0 дней для записи).

## Ограничение запросов

Ограничение запросов применяется по API-ключу с помощью бакета скользящего окна в `SearchRateLimitBucket`.

По умолчанию: 60 запросов в минуту на ключ (настраивается для каждого ключа). При превышении лимита возвращается HTTP 429
с `error: "rate_limit_exceeded"` и заголовком `Retry-After`.

## Разрешение плана

Активный план организации разрешается через `resolveOrgPlan(orgId)` в `@repo/payments`:

1. Поиск последней `Purchase` (подписки) для организации.
2. Сопоставление `priceId` → `planId` с использованием матрицы цен в `packages/payments/config.ts`.
3. Кэш на 60 секунд (`invalidatePlanCache(orgId)` вызывается при событиях подписки).
4. Fail-open: если разрешение не удалось (платёжный провайдер недоступен), по умолчанию применяются ограничения бесплатного плана.

## Процедура прав доступа oRPC

Процедура `entitlements.getPlanInfo` возвращает текущий план организации и сводку использования:

```typescript
const planInfo = await orpc.entitlements.getPlanInfo.call({ organizationId });
// Возвращает: { planId, planName, searchUnitsUsed, searchUnitsLimit, indexCount, indexLimit, features }
```

## Процесс обновления плана

Текущий статус: **Частичный.** Stripe-биллинг для поисковых единиц запланирован для v0.6. Tochka (российский
платёжный провайдер) активно подключается в `packages/payments/provider/tochka-wallet/`.

Компонент `BillingPlanInfo` в `apps/saas/modules/search/components/BillingPlanInfo.tsx` отображает
текущий план и использование. CTA обновления ссылается на страницу настроек биллинга.

## Переопределения администратора

Администраторы организации могут просматривать информацию о плане в панели управления в **Settings → Billing**. Платформенные
администраторы (через `/admin/organizations`) могут проверять права доступа любой организации и вручную корректировать
назначение плана в базе данных в исключительных случаях.
