Данная страница является авторитетным справочником по статусу функций. Используйте её, чтобы понять, на что можно опираться в production, что активно разрабатывается и что остаётся в дорожной карте.

## Готово (production-ready)

Эти функции существуют в кодовой базе и готовы к использованию.

### Основной поиск

| Функция                                      | Пакет / файл                          |
| -------------------------------------------- | ------------------------------------- |
| Клиент Typesense (только на стороне сервера) | `packages/search/lib/client.ts`       |
| Управление схемой коллекции + версионирование | `packages/search/lib/collections.ts` |
| Буфер приёма данных через БД                 | `packages/search/lib/buffer.ts`       |
| searchDocuments / multiSearchDocuments       | `packages/search/lib/search.ts`       |
| Переиндексация с атомарной заменой псевдонима | `packages/search/lib/reindex.ts`     |
| Генерация и хэширование API-ключей           | `packages/search/lib/keys.ts`         |
| Браузерный SDK                               | `packages/search-client/src/index.ts` |

### Аутентификация и мультиарендность

| Функция                                              | Примечания                                                       |
| ---------------------------------------------------- | ---------------------------------------------------------------- |
| Аутентификация с областью организаций через Better Auth | Организации — основное рабочее пространство                  |
| API-ключи (хэшированные, ограниченные, с ограничением origin) | Префиксы: `ss_search_*`, `ss_connector_*`, `ss_scoped_*` |
| Ограниченные поисковые токены (HMAC)                 | Stateless, подписаны через `BETTER_AUTH_SECRET`                  |
| Применение квот по плану                             | `packages/api/modules/entitlements/`                             |
| Ограничение запросов по ключу/IP                     | `packages/search/lib/keys.ts` + бакет в БД                      |

### Виджет и коннектор

| Функция                                  | Примечания                                                                                   |
| ---------------------------------------- | -------------------------------------------------------------------------------------------- |
| Хостинговый виджет                       | `packages/widget/` — Vanilla JS, Shadow DOM, 14 КБ                                          |
| Connector API                            | 7 эндпоинтов: handshake, heartbeat, sync.full, sync.delta, delete, diagnostics, sync/status |
| CRUD токенов коннектора                  | `searchRouter.{createConnectorToken, listConnectorTokens, revokeConnectorToken}`             |
| Отслеживание заданий синхронизации (in-memory) | Теряется при перезапуске — приемлемо для MVP                                          |

### Управление релевантностью

| Функция              | Примечания                            |
| -------------------- | ------------------------------------- |
| Синонимы             | `searchRouter.synonyms.{get,update}`  |
| Кураторство          | `searchRouter.curations.{get,update}` |
| Управление схемой коллекции | `searchRouter.schema.{get,update}` |

### Модуль знаний (R2.5)

RAG/GraphRAG для работы с документами. Отдельная продуктовая поверхность от поиска витрины.

- Модели Prisma: `KnowledgeSpace`, `DataSource`, `IngestionJob`, `KnowledgeDocument`, `KnowledgeChunk`, `GraphNode`, `GraphEdge`
- 10 процедур oRPC: list/create spaces, list/create sources, list ingestion jobs, ingest-file, ask, graphrag-explain, usage-metrics
- UI `KnowledgeWorkbench.tsx`, маршруты на уровне организации + на уровне аккаунта

## В активной разработке

Эти функции находятся в активной разработке и могут изменяться.

| Функция                                                   | Статус                                                               |
| --------------------------------------------------------- | -------------------------------------------------------------------- |
| Тарификация v0.6 (пополнение Tochka, кошелёк)             | `packages/billing-wallet/`, `packages/payments/provider/tochka-wallet/` |
| Агрегация `recordSearchUsage` rollup                      | Сырые строки существуют; таблица `UsageCounter` отложена             |
| Административные страницы (аудит, кошелёк, уведомления, безопасность) | Зафиксированы как заглушки — оперативный контент уточняется |

## Скелет / не отслеживается

| Функция           | Примечания                                                         |
| ----------------- | ------------------------------------------------------------------ |
| Модуль PrestaShop | `modules/prestashop/aacsearch/` — PHP, отдельный трек коммитов     |
| Модуль Bitrix     | `modules/bitrix/aac.search/` — PHP, отдельный трек коммитов        |

Эти модули имеют корректную структуру файлов (`Aacsearch.php`, `config.xml`, `classes/`, `controllers/`, `views/`),
но не отслеживаются в основном репозитории. Они следуют контракту Connector API, описанному в
[Жизненный цикл Connector API](/docs/connectors/connector-api-lifecycle).

## Вне области видимости (v1)

Следующие функции **явно не запланированы** для v1:

- AI-чат / разговорный поиск
- Профили персонализации на посетителя
- Движок рекомендаций
- A/B-тестирование релевантности
- Enterprise SSO (SAML/OIDC)
- Мультирегиональная репликация Typesense
- Выделенные одноарендные кластеры
- Коннекторы помимо PrestaShop и Bitrix
- Глубокая BI-аналитика за рамками карточек дашборда
