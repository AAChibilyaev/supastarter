AACsearch использует синтаксис фильтров и сортировки Typesense с несколькими соглашениями, специфичными для
схемы товаров. Данная страница является полным справочником по построению выражений фильтров, настройке
порядка сортировки и постраничной навигации по результатам.

## Синтаксис выражений фильтров

Фильтры задаются строкой с булевой логикой.

### Равенство

```
availability:=in_stock
brand:=Sony
categories:=Electronics
locale:=en
```

### Отрицание

```
availability:!=out_of_stock
brand:!=Competitors
```

### Диапазон (числовой)

```
price:>50
price:<200
price:[50..200]     # включительный диапазон
price:(50..200)     # исключительный диапазон
```

### Принадлежность массиву

```
categories:=[Electronics, Audio]
brand:=[Sony, Bose, Sennheiser]
```

### Строка содержит / префикс

Typesense поддерживает только точное совпадение строк для полей с фасетами. Для поиска по префиксу используйте `q` с `queryBy`.

### Булева логика

```
# AND (оба условия должны совпадать)
availability:=in_stock && price:<200

# OR (любое из условий совпадает)
brand:=Sony || brand:=Bose

# Группировка
(brand:=Sony || brand:=Bose) && availability:=in_stock && price:<200
```

### Фильтр арендатора (автоматический)

Публичный обработчик автоматически добавляет:

```
organization_id:={organizationId}
```

Этот фильтр добавлять вручную не нужно — он всегда применяется.

## Доступные поля фильтрации

Схема товаров по умолчанию включает следующие поля для фильтрации (помечены `facet: true`):

| Поле           | Тип      | Пример фильтра                     |
| -------------- | -------- | ---------------------------------- |
| `brand`        | string   | `brand:=Sony`                      |
| `categories`   | string[] | `categories:=[Electronics, Audio]` |
| `availability` | string   | `availability:=in_stock`           |
| `price`        | float    | `price:[10..500]`                  |
| `sale_price`   | float    | `sale_price:<100`                  |
| `locale`       | string   | `locale:=en`                       |

## Фасетирование

Фасеты вычисляют распределение значений поля по совпадающим документам. Используйте их для рендера боковых панелей фильтров.

```json
{
	"q": "headphones",
	"facetBy": "brand,categories,availability"
}
```

Ответ содержит `facetCounts`:

```json
{
	"facetCounts": [
		{
			"fieldName": "brand",
			"counts": [
				{ "value": "Sony", "count": 45 },
				{ "value": "Bose", "count": 28 },
				{ "value": "Sennheiser", "count": 19 }
			]
		},
		{
			"fieldName": "availability",
			"counts": [
				{ "value": "in_stock", "count": 89 },
				{ "value": "out_of_stock", "count": 23 }
			]
		}
	]
}
```

### Фасеты с применёнными фильтрами

Когда активен фильтр, счётчики фасетов отражают отфильтрованный набор результатов, а не весь индекс:

```json
{
	"q": "*",
	"filterBy": "brand:=Sony",
	"facetBy": "categories,availability"
}
```

Возвращает количества категорий и наличие только для товаров Sony.

## Сортировка

### Сортировка по одному полю

```json
{ "sortBy": "price:asc" }
{ "sortBy": "price:desc" }
{ "sortBy": "created_at:desc" }
{ "sortBy": "sale_price:asc" }
```

### Сортировка по релевантности (по умолчанию)

```json
{ "sortBy": "_text_match:desc" }
```

Это значение по умолчанию, когда `sortBy` не указан.

### Многопольная сортировка (тайбрейкер)

```json
{ "sortBy": "_text_match:desc,price:asc" }
```

Сортирует сначала по оценке релевантности; при одинаковых оценках — по цене по возрастанию.

### Типичные пресеты сортировки

| Метка                           | Значение `sortBy`  |
| ------------------------------- | ------------------ |
| Наиболее релевантные            | `_text_match:desc` |
| Цена: сначала низкая            | `price:asc`        |
| Цена: сначала высокая           | `price:desc`       |
| Сначала новые                   | `created_at:desc`  |
| Цена со скидкой: сначала низкая | `sale_price:asc`   |

## Пагинация

```json
{
	"page": 1, // с 1, по умолчанию: 1
	"perPage": 20 // по умолчанию: 10, макс.: 100
}
```

Поля ответа для пагинации:

```json
{
	"found": 142, // всего совпадающих документов
	"page": 1, // текущая страница
	"outOf": 5000 // всего документов в индексе (без фильтрации)
}
```

Вычисление общего числа страниц:

```javascript
const totalPages = Math.ceil(results.found / perPage);
const hasMore = results.page * perPage < results.found;
```

### Предупреждение о глубокой пагинации

Производительность Typesense снижается при больших значениях `page` для индексов более 100К документов.
Для глубокой пагинации или экспортов:

1. Используйте возрастающие значения `page` с фиксированным `perPage`
2. Или реализуйте пагинацию на основе курсора, сортируя по уникальному полю (`id:asc`)
   и используя `filterBy: "id:>lastSeenId"`

## Выбор полей

Уменьшите размер ответа, выбрав только нужные поля:

```json
{
	"includeFields": "id,title,price,brand,availability",
	"excludeFields": "description,tags"
}
```

Используйте `excludeFields` для удаления больших текстовых полей из ответов, когда вы их не рендерите
(например, `description` в автодополнении поиска по мере ввода, которое показывает только заголовки).

## Комбинирование фильтров с ограниченными токенами

При использовании ограниченного токена его `scopedFilter` объединяется через AND с `filterBy` вызывающей стороны:

```
scopedFilter токена: "price:<100"
filterBy вызывающей стороны: "brand:=Sony && availability:=in_stock"
Эффективный filterBy: "brand:=Sony && availability:=in_stock && price:<100"
```

Фильтр токена не может быть переопределён `filterBy` вызывающей стороны.
