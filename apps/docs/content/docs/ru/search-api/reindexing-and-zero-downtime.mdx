---

AACsearch использует стратегию **версионированная коллекция + псевдоним** для выполнения миграций схемы и полных
переиндексаций без простоя поиска. Запросы всегда направляются к псевдониму (стабильному имени), тогда как
версионированные коллекции меняются под ним.

## Модель с атомарной заменой псевдонима

Каждый поисковый индекс использует два имени Typesense:

```
псевдоним:  org123_products        ← стабильный, используется всеми поисковыми запросами
коллекция:  org123_products_v1     ← текущая версионированная коллекция (обслуживает трафик)
```

При запуске переиндексации:

```
1. СОЗДАТЬ  org123_products_v2       ← новая коллекция с обновлённой схемой
2. ИМПОРТ   все документы → v2       ← повторный приём данных из БД (или снимка)
3. ПРОВЕРИТЬ количество документов(v2) ≈ ожидаемое ← проверка работоспособности перед заменой
4. ЗАМЕНИТЬ псевдоним: org123_products → org123_products_v2
5. СОХРАНИТЬ org123_products_v1     ← сохраняется до следующей переиндексации (вариант отката)
```

Шаг 4 (замена псевдонима) атомарен в Typesense. Запросы продолжают попадать в старую версию во время шагов
1–3 и беспрепятственно переходят к новой версии после шага 4 — без какого-либо простоя.

## Когда требуется переиндексация

| Сценарий                              | Переиндексация требуется?                     |
| ------------------------------------- | --------------------------------------------- |
| Добавлено новое поле-фасет            | **Да** — Typesense требует миграции схемы     |
| Изменён тип поля                      | **Да**                                        |
| Добавлено новое текстовое поле поиска | **Да**                                        |
| Удалено поле                          | **Да**                                        |
| Обновлено содержимое документа        | Нет — используйте upsert/дельта-синхронизацию |
| Изменены правила синонимов            | Нет — синонимы являются отдельным ресурсом    |
| Изменены параметры сортировки         | Нет — `sortBy` является параметром запроса    |
| Изменены правила кураторства          | Нет — кураторство является отдельным ресурсом |

## Запуск переиндексации

### Через панель управления

1. Перейдите в **Search** → **Indexes** → выберите индекс
2. Нажмите кнопку **Reindex** в настройках индекса
3. Отслеживайте прогресс на панели заданий импорта

### Через oRPC

```typescript
const job = await orpc.search.reindex.call({
	organizationId: "org_...",
	indexSlug: "products",
});
// Возвращает: { jobId, status: "queued" }
```

### Через Connector API (admin)

```bash
POST /api/v1/indexes/{indexSlug}/reindex
Authorization: Bearer aa_admin_your_key
```

## Мониторинг прогресса переиндексации

Проверьте статус задания импорта:

```typescript
const jobs = await orpc.search.importJobs.call({
	organizationId: "org_...",
	indexSlug: "products",
});

const activeJob = jobs.find((j) => j.type === "reindex" && j.status === "running");
// activeJob.processedItems / activeJob.totalItems → прогресс
```

В панели управления вкладка **Import Jobs** показывает прогресс переиндексации с индикатором прогресса.

## Обновления схемы

Перед переиндексацией обновите схему коллекции:

```typescript
await orpc.search.schema.update.call({
	organizationId: "org_...",
	indexSlug: "products",
	fields: [
		// Добавить новое поле
		{ name: "tags", type: "string[]", facet: true, optional: true },
	],
});
```

Это обновляет определение схемы, хранящееся в AACsearch. Фактическая схема коллекции Typesense
обновляется во время следующей переиндексации (изменения схемы не применяются на месте во избежание простоя).

## Влияние на квоту

Переиндексация повторно принимает все документы. Для каталога из 50К товаров переиндексация потребляет 50К поисковых единиц.
Планируйте расписание переиндексации с учётом вашей ежемесячной квоты.

## Откат

Если новая коллекция имеет проблемы, выполните откат, перенаправив псевдоним обратно на предыдущую версию:

```typescript
await orpc.search.reindex.call({
	organizationId: "org_...",
	indexSlug: "products",
	rollback: true, // переключает псевдоним обратно на предыдущую версию
});
```

Предыдущая версия (`v{N-1}`) сохраняется до следующей успешной переиндексации.

## Детали реализации

Реализация переиндексации находится в `packages/search/lib/reindex.ts` (экспортируется как `reindexCollection`).

Ключевые особенности поведения:

- Версионированное имя коллекции: `{orgShortId}_{slug}_v{nextVersion}`
- Замена псевдонима: `typesense.aliases.upsert(alias, { collection_name: newCollectionName })`
- Источник документов: повторное дренирование полного набора документов из буфера приёма или снимка
- Частичный сбой: если импорт не удался до замены, старая версия продолжает обслуживать запросы; новая версия удаляется

Функция вызывается из процедуры `searchRouter.reindex` в `packages/api/modules/search/procedures/reindex.ts`.
