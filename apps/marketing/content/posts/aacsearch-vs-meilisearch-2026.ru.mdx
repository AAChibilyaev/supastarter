Если вы оцениваете провайдеров search-as-a-service в 2026 году, AACsearch и MeiliSearch Cloud наверняка в вашем шорт-листе. Оба предлагают hosted-поиск с Typesense-подобной простотой, но принципиально по-разному подходят к ценообразованию, мультитенантности и аналитике.

Это сравнение охватывает реальные различия — а не маркетинговые заявления.

## Краткий обзор

| Возможность       | AACsearch                                   | MeiliSearch Cloud                   |
| ----------------- | ------------------------------------------- | ----------------------------------- |
| Движок            | Typesense (open source)                     | MeiliSearch (open source)           |
| Хостинг           | Managed (мультирегион)                      | Managed (US/EU)                     |
| Мультитенантность | Встроенная — scoped API tokens              | Не встроена — нужен proxy-слой      |
| API-ключи         | 3 типа: search, admin, scoped               | Управление API-ключами (базовое)    |
| Search Widget     | Включён (script-тег, 35KB)                  | Instant MeiliSearch (React-heavy)   |
| Аналитика         | Встроенная — бесплатно на всех планах       | Минимальная (open-source dashboard) |
| Геопоиск          | Встроенная поддержка                        | Не встроен                          |
| Допуск опечаток   | Настраивается поиндексно                    | По умолчанию (мин/макс опечатки)    |
| Синонимы          | UI-дашборд + массовый импорт                | Дашборд + API                       |
| Коннекторы        | PrestaShop, Bitrix, WooCommerce, Shopify    | Нет                                 |
| Векторный поиск   | Да (embedding-поля)                         | Да (гибридный поиск)                |
| Self-host опция   | Полный экспорт Typesense                    | Да (open source)                    |
| SLA               | 99.9% (Scale+)                              | 99.95% (Cloud)                      |
| Бесплатный тариф  | 1 индекс, 10K документов, безлимит операций | 1 индекс, лимитированные операции   |

## Сравнение цен

AACsearch взимает плату за индекс с **неограниченным количеством поисковых операций**. MeiliSearch Cloud взимает плату за каждую поисковую операцию — чем больше ищут ваши пользователи, тем больше вы платите.

### Сценарий 1: Небольшой e-commerce (10K товаров, 50K поисков/мес)

| Компонент затрат   | AACsearch             | MeiliSearch Cloud                |
| ------------------ | --------------------- | -------------------------------- |
| Индекс/тариф       | Бесплатный тариф      | $29/мес (Starter)                |
| Поисковые операции | Безлимитно (включено) | ~$5–10 (превышение)              |
| Аналитика          | Включена              | Дополнительно (если self-hosted) |
| **Итого**          | **$0/мес**            | **~$35–40/мес**                  |

### Сценарий 2: Mid-market B2B SaaS (500K документов, 1M поисков/мес)

| Компонент затрат   | AACsearch                | MeiliSearch Cloud                    |
| ------------------ | ------------------------ | ------------------------------------ |
| Индекс/тариф       | $99/мес (Scale)          | $399/мес (Business)                  |
| Поисковые операции | Безлимитно (включено)    | ~$100 (превышение при 1M)            |
| Мультитенантность  | Включена (scoped tokens) | Дополнительно (proxy-инфраструктура) |
| **Итого**          | **$99/мес**              | **~$500/мес**                        |

### Сценарий 3: Крупный SaaS (5M документов, 10M поисков/мес)

| Компонент затрат     | AACsearch             | MeiliSearch Cloud                          |
| -------------------- | --------------------- | ------------------------------------------ |
| Индекс/тариф         | $249/мес (Pro)        | Индивидуально (ориент. $1.5–2K/мес)        |
| Поисковые операции   | Безлимитно (включено) | Индивидуально (ориент. $500–1K превышение) |
| Выделенная инфрастр. | Доступна              | Недоступна                                 |
| **Итого**            | **$249/мес**          | **~$2–3K/мес**                             |

**Ключевое различие в ценообразовании:** Поиндексная модель AACsearch делает затраты предсказуемыми. Пооперационная модель MeiliSearch Cloud означает, что ваш счёт растёт вместе с объёмом поиска — проблема масштабирования для успешных продуктов.

## Мультитенантность: побеждает AACsearch

Для SaaS-платформ, обслуживающих несколько организаций, мультитенантность — решающий фактор.

**AACsearch** имеет встроенные scoped API-токены. Создайте один ключ на каждого арендатора с фильтром вида `tenant_id:=:acme_corp`. Каждый запрос с этим токеном автоматически фильтруется. Никакого middleware, прокси или риска утечки данных между арендаторами.

**MeiliSearch Cloud** не имеет встроенной мультитенантности. Вам придётся реализовать proxy-слой, который добавляет фильтры арендатора к каждому запросу. Это означает:

- Дополнительная задержка (запрос проходит через ваш сервер)
- Дополнительная инфраструктура для обслуживания
- Риск утечки данных при неправильной настройке прокси

Для SaaS-приложения с 200 арендаторами только инфраструктура прокси добавляет $50–200/мес к затратам на серверы.

## Аналитика: побеждает AACsearch

Поисковая аналитика критически важна для улучшения релевантности. AACsearch включает её в каждый тариф — объём поиска, популярные запросы, рейтинг кликов, отчёты по нулевым результатам и отслеживание конверсий.

Аналитика MeiliSearch Cloud минимальна. Open-source дашборд MeiliSearch предоставляет базовые метрики, но для production-grade аналитики вам понадобятся дополнительные инструменты (PostHog, Mixpanel или собственная аналитическая система). Это добавляет накладные расходы на интеграцию и обслуживание.

## Геопоиск: побеждает AACsearch

Если ваше приложение имеет какие-либо функции, зависящие от местоположения (поиск магазинов, «рядом со мной», локальный инвентарь), геопоиск — это минимальное требование.

AACsearch поддерживает геопоиск нативно с фильтрацией по радиусу:

```json
{
	"q": "coffee shops",
	"filter_by": "_geo_dist(40.7128, -74.0060, 5000)"
}
```

MeiliSearch Cloud не имеет встроенного геопоиска. Вам придётся реализовывать его через отдельный геокодинг-сервис и фильтровать вручную — это добавляет сложность и задержку.

## Widget и фронтенд: Ничья

Обе платформы предлагают UI-компоненты для поиска, но с разными подходами:

| Аспект                | AACsearch Widget        | MeiliSearch Instant MeiliSearch    |
| --------------------- | ----------------------- | ---------------------------------- |
| Установка             | 3-строчный script-тег   | React-компонент + шаг сборки       |
| Размер бандла         | 35KB JS + 8KB CSS       | ~60KB (React-heavy)                |
| Поддержка фреймворков | Любая HTML-страница     | Только React (или кастомный)       |
| Командная палитра     | Включена (Cmd+K режим)  | Доступна через плагин              |
| Фасетные фильтры      | Встроенные, мультивыбор | Встроенные                         |
| Темизация             | CSS-переменные          | CSS + переопределения через плагин |

Виджет AACsearch работает на любом сайте — статический HTML, WordPress, Astro или React. Instant MeiliSearch от MeiliSearch работает только с React, что исключает статические сайты и фреймворки, не основанные на React.

## Коннекторы: побеждает AACsearch

AACsearch поставляется с готовыми коннекторами для основных e-commerce-платформ: PrestaShop, Bitrix24, WooCommerce и Shopify (в разработке). Они обеспечивают авто-маппинг схем, инкрементальную синхронизацию и распространение удалений.

MeiliSearch Cloud не предлагает готовых коннекторов. Каждая интеграция — это кастомная реализация.

## Векторный поиск: поддерживают обе платформы

Обе платформы поддерживают векторные эмбеддинги для семантического поиска:

| Возможность          | AACsearch                   | MeiliSearch Cloud                  |
| -------------------- | --------------------------- | ---------------------------------- |
| Реализация           | Embedding-поля в документах | Гибридный поиск (keyword + vector) |
| Источник эмбеддингов | Свои эмбеддинги             | Свои эмбеддинги                    |
| Варианты применения  | Семантический поиск, RAG    | Семантический поиск, рекомендации  |

AACsearch использует механизм эмбеддингов Typesense, где вы включаете векторные эмбеддинги как поля документов. MeiliSearch Cloud поддерживает гибридный поиск, сочетающий keyword и векторное сопоставление.

## Когда что выбирать

### Выбирайте AACsearch, когда:

- Вам нужна мультитенантность (scoped API-токены для каждой организации)
- Вы хотите предсказуемое поиндексное ценообразование (не пооперационную оплату)
- Важна встроенная аналитика (отслеживание нулевых результатов, CTR, конверсий)
- Вам нужен геопоиск (поиск магазинов, «рядом со мной»)
- Вы используете статический сайт или фреймворк не на React
- Вам нужны готовые CMS-коннекторы
- Важен экспорт данных / отсутствие блокировки (zero lock-in)

### Выбирайте MeiliSearch Cloud, когда:

- Вы уже используете open-source экосистему MeiliSearch
- Вам нужны специфические функции MeiliSearch (federated search)
- Вы хотите self-hosted решение (MeiliSearch полностью open source)
- Вы предпочитаете UI-компоненты на React/InstantSearch
- Вам требуется SLA 99.95% как жёсткое требование
- Вы создаёте функциональность только для поиска (без потребности в мультитенантности)

## Путь миграции

Переход с MeiliSearch на AACsearch:

1. Экспортируйте дамп MeiliSearch
2. Уплостите вложенные документы (Typesense предпочитает плоские схемы)
3. Импортируйте в AACsearch через bulk API
4. Настройте scoped API-токены на каждого арендатора (недоступно в MeiliSearch)
5. Настройте формулу ранжирования и фасетные атрибуты в дашборде AACsearch
6. Разверните виджет AACsearch (замените Instant MeiliSearch)

Миграция обычно занимает 2–4 часа для индекса среднего размера.

## Вердикт

AACsearch — лучший выбор для большинства production SaaS-приложений, особенно тех, у которых несколько арендаторов, есть потребность в аналитике или требования к геопоиску. MeiliSearch Cloud — надёжный вариант для однопользовательских проектов, уже инвестировавших в экосистему MeiliSearch.

Для наиболее распространённого случая — B2B SaaS-приложения, которому нужен поиск — встроенная мультитенантность AACsearch и ценообразование с неограниченными операциями экономят как время на разработку, так и текущие эксплуатационные расходы.
