La recherche est la fonctionnalité la plus impactante que vous puissiez ajouter à un site web — et avec le Widget AACsearch, c'est aussi la plus simple. Trois lignes de HTML transforment n'importe quel site en une expérience de recherche avec autocomplétion, navigation au clavier et filtrage à facettes.

## La Configuration en 3 Lignes

Ajoutez ces trois lignes à n'importe quelle page HTML :

```html
<script src="https://cdn.aacsearch.com/widget.js"></script>
<link rel="stylesheet" href="https://cdn.aacsearch.com/widget.css" />
<div id="search-root"></div>
```

Ensuite, dans une balise script ou votre fichier JavaScript principal :

```javascript
new AACsearchWidget({
	apiKey: "ss_search_your_api_key",
	index: "products",
	mountElement: "#search-root",
});
```

Voilà. Votre site dispose désormais d'une barre de recherche complète qui retourne des résultats au fur et à mesure que l'utilisateur tape.

## Comment Ça Fonctionne

Le Widget se connecte directement à l'API AACsearch depuis le navigateur. Lorsqu'un utilisateur tape, le Widget :

1. Envoie la requête à AACsearch avec votre clé API et votre index
2. Reçoit les résultats correspondants (généralement en moins de 50 ms)
3. Affiche une superposition avec des fiches produit, des extraits ou des modèles personnalisés
4. Gère la navigation au clavier (flèches, Entrée, Échap)
5. Suit les événements de clic pour les analytics

Pas de serveur backend, pas d'étape de build, pas de synchronisation de base de données. Le Widget gère chaque interaction.

## Ce Que Vous Obtenez Prêt à l'Emploi

Le Widget est livré avec ces fonctionnalités actives :

| Fonctionnalité        | Description                                                                   |
| --------------------- | ----------------------------------------------------------------------------- |
| Autocomplétion        | Les résultats apparaissent en tapant, avec un délai de 300 ms                 |
| Navigation clavier    | Flèches pour naviguer, Entrée pour sélectionner, Échap pour fermer            |
| Tolérance aux fautes  | Les requêtes mal orthographiées retournent des résultats pertinents           |
| Extraits de résultats | Affiche le titre, l'extrait, la vignette et le prix (configurable)            |
| Mode sombre           | Détection automatique des préférences système, variables CSS personnalisables |
| Adaptatif mobile      | Pleine largeur sur mobile, tiroir de facettes en bas sur petits écrans        |
| Analytics             | Suit les requêtes de recherche, les clics et les recherches sans résultat     |
| Filtres à facettes    | Affiche les panneaux de facettes lorsque les champs sont configurés           |

## Exemples d'Intégration

### Site HTML Statique

```html
<!DOCTYPE html>
<html>
	<head>
		<script src="https://cdn.aacsearch.com/widget.js" defer></script>
		<link rel="stylesheet" href="https://cdn.aacsearch.com/widget.css" />
	</head>
	<body>
		<header>
			<div id="search-root"></div>
		</header>
		<script>
			document.addEventListener("DOMContentLoaded", () => {
				new AACsearchWidget({
					apiKey: "ss_search_demo_key",
					index: "docs",
					placeholder: "Search documentation...",
					mountElement: "#search-root",
				});
			});
		</script>
	</body>
</html>
```

### WordPress (functions.php)

```php
function add_aacsearch_widget() {
    wp_enqueue_script('aacsearch-widget', 'https://cdn.aacsearch.com/widget.js', [], null, true);
    wp_enqueue_style('aacsearch-widget', 'https://cdn.aacsearch.com/widget.css');
}
add_action('wp_enqueue_scripts', 'add_aacsearch_widget');

function render_aacsearch_widget() {
    echo '<div id="search-root"></div>';
    echo '<script>
        document.addEventListener("DOMContentLoaded", function() {
            new AACsearchWidget({
                apiKey: "' . esc_js(get_option('aacsearch_api_key')) . '",
                index: "wordpress_posts",
                mountElement: "#search-root"
            });
        });
    </script>';
}
add_action('wp_footer', 'render_aacsearch_widget');
```

### React / Next.js

```tsx
"use client";

import { useEffect, useRef } from "react";

export function AACSearchWidget() {
	const ref = useRef<HTMLDivElement>(null);

	useEffect(() => {
		if (!ref.current) return;

		const widget = new AACsearchWidget({
			apiKey: process.env.NEXT_PUBLIC_AACSEARCH_API_KEY!,
			index: "blog",
		});

		widget.mount(ref.current);
		return () => widget.destroy();
	}, []);

	return <div ref={ref} />;
}
```

### Vue.js

```vue
<template>
	<div ref="searchRoot"></div>
</template>

<script setup>
import { ref, onMounted, onUnmounted } from "vue";

const searchRoot = ref(null);
let widget = null;

onMounted(() => {
	widget = new AACsearchWidget({
		apiKey: "ss_search_demo_key",
		index: "products",
		mountElement: searchRoot.value,
	});
});

onUnmounted(() => widget?.destroy());
</script>
```

## Configuration Avancée

Le Widget accepte ces options pour un contrôle précis :

```javascript
const search = new AACsearchWidget({
	// Required
	apiKey: "ss_search_...",
	index: "products",

	// Display
	placeholder: "Search products...",
	theme: "light", // 'light' | 'dark' | 'auto'
	resultsPerPage: 12,
	maxSuggestions: 5,

	// Search behavior
	searchableFields: ["name", "brand", "description", "sku"],
	facetFields: ["brand", "category", "price"],
	sortOptions: [
		{ label: "Relevance", value: "_text_match:desc" },
		{ label: "Price: Low to High", value: "price:asc" },
	],

	// Custom rendering
	resultTemplate: (item) => `
    <div class="aacsearch-result">
      <img src="${item.image}" alt="${item.name}" />
      <div class="aacsearch-result-info">
        <h3>${item.name}</h3>
        <p class="price">$${item.price}</p>
        <span class="brand">${item.brand}</span>
      </div>
    </div>
  `,

	// Analytics
	trackClicks: true,
	onSearch: (query, results) => console.log("Searched:", query, "Results:", results.length),
	onClick: (result) => console.log("Clicked:", result.name),
});
```

## Performance

- **Premier rendu :** < 200 ms (le JS du Widget fait ~35 KB gzippé)
- **Temps de réponse des requêtes :** < 50 ms (P95 depuis la côte ouest américaine)
- **Rendu des résultats :** < 20 ms (défilement virtuel pour les grands ensembles de résultats)
- **Taille du bundle :** 35 KB JS + 8 KB CSS (gzippé)

## Limitations

Le Widget est conçu pour la recherche — pas pour construire une vitrine complète. Utilisez-le pour :

- Recherche sur site de documentation (meilleur usage)
- Recherche de produits e-commerce
- Recherche de blog / contenu
- Palette de commandes d'application SaaS

Pour les interfaces complexes (pages produit personnalisées, flux de paiement), utilisez directement l'API REST AACsearch.
