PrestaShop wird mit einer einfachen Suchmaschine geliefert. Sie funktioniert für einfache Shops, scheitert aber schnell: keine Tippfehlertoleranz, kein Präfix-Matching, keine Echtzeit-Updates und ein Ranking, das Conversion-Signale wie Verkaufsrang oder Marge ignoriert.

Wenn Kunden Ihren Shop verlassen, weil sie keine Produkte finden können, oder wenn Ihre „Keine Ergebnisse"-Rate über 10 % liegt, ist es Zeit für ein Upgrade.

Dieses Tutorial führt Sie durch das Hinzufügen von AACSearch zu einem PrestaShop-Store — von der Modul-Installation bis zum ersten Live-Suchergebnis.

## Was AACSearch zu PrestaShop hinzufügt

Bevor wir zur Einrichtung kommen, hier ist, was sich ändert, wenn Sie AACSearch installieren:

- **Sofortergebnisse während der Eingabe** — Ergebnisse erscheinen innerhalb von 30–50ms, bevor der Kunde seine Anfrage abgeschlossen hat
- **Tippfehlertoleranz** — „iphne 15" findet iPhone 15; „swartz kleid" findet schwarze Kleider
- **Präfix-Matching** — „kabel" trifft „Kabelsalat", „Kabeltrommel" usw.
- **Automatische Synchronisation** — Preisänderungen, neue Produkte und Lageraktualisierungen erscheinen innerhalb von Minuten in der Suche
- **Benutzerdefiniertes Ranking** — Ranking nach Umsatz, Marge, Lagerbestand oder einem beliebigen benutzerdefinierten Feld

Die native PrestaShop-Suche macht nichts davon zuverlässig.

## Voraussetzungen

- PrestaShop 8.x (7.x teilweise unterstützt)
- Ein AACSearch-Konto — [kostenlos anmelden](/)
- Admin-Zugang zum PrestaShop-Backend
- FTP- oder Dateimanager-Zugang (für manuellen Upload, falls Marketplace nicht verfügbar ist)

## Schritt 1: Erstellen Sie Ihren AACSearch-Index

Melden Sie sich in Ihrem AACSearch-Dashboard an. Klicken Sie auf **Neuer Index** und geben Sie ihm einen Namen (z. B. `products`). Ihr Index wird sofort bereitgestellt — keine Konfiguration in diesem Stadium erforderlich.

## Schritt 2: Generieren Sie ein Connector-Token

Gehen Sie im AACSearch-Dashboard zu **Connectoren** in der linken Seitenleiste. Klicken Sie auf **Neues Token**. Geben Sie ihm einen Namen (z. B. „PrestaShop Produktion") und kopieren Sie das Token — Sie benötigen es in Schritt 3.

Halten Sie dieses Token privat. Es hat Schreibzugriff auf Ihren Index und sollte niemals in clientseitigem Code oder öffentlichen Repositories erscheinen.

## Schritt 3: Installieren Sie das AACSearch-Modul

### Über den PrestaShop Marketplace (empfohlen)

Gehen Sie in Ihrem PrestaShop-Backend zu **Modulverwaltung** → **Marketplace** und suchen Sie nach „AACSearch". Klicken Sie auf **Installieren**.

### Über ZIP-Upload

Laden Sie das Modul-ZIP von Ihrem AACSearch-Dashboard herunter (unter **Connectoren** → **Modul herunterladen**). Gehen Sie im Backend zu **Modulverwaltung** → **Modul hochladen** und laden Sie das ZIP hoch.

Nach der Installation sehen Sie die AACSearch-Konfigurationsseite. Geben Sie ein:

- **Connector-Token**: Fügen Sie das Token aus Schritt 2 ein
- **Index-Name**: Geben Sie den Index-Namen ein, den Sie in Schritt 1 erstellt haben (z. B. `products`)

Klicken Sie auf **Speichern und Verbindung testen**. Sie sollten eine grüne Erfolgsmeldung mit Ihrem Organisationsnamen und der Dokumentanzahl des Index sehen.

## Schritt 4: Führen Sie die erste Synchronisation durch

Klicken Sie auf der Konfigurationsseite auf **Vollständige Synchronisation**. Dadurch wird Ihr gesamter Produktkatalog — Namen, Beschreibungen, Preise, Kategorien, Bilder, Attribute — exportiert und in Ihren Index übertragen.

Die Synchronisation läuft im Hintergrund. Für einen Katalog mit 10.000 Produkten dauert es typischerweise 1–3 Minuten. Sie können den Fortschritt im Tab **Sync-Jobs** Ihres Dashboards verfolgen.

## Schritt 5: Delta-Synchronisation konfigurieren

Vollständige Synchronisationen sind für die erste Befüllung. Für laufende Updates aktivieren Sie in den Moduleinstellungen **Automatische Synchronisation**. Das Standardintervall beträgt 5 Minuten — der Connector prüft auf Änderungen seit der letzten Synchronisation und sendet nur diese.

## Schritt 6: Das Such-Widget einbetten

### Automatisch (über Modul-Hook)

Aktivieren Sie in der Konfiguration **Standard-Suche ersetzen**. Das Modul fügt das Widget automatisch hinzu — keine Template-Bearbeitung erforderlich.

### Manuell (für benutzerdefinierte Themes)

Kopieren Sie den Widget-Snippet aus dem Dashboard (Tab **Widget**):

```html
<script
	src="https://app.aacsearch.com/api/widget/widget.js"
	data-index-id="IHRE_INDEX_ID"
	data-api-key="IHR_SUCH_KEY"
	data-locale="{language.iso_code}"
	async
></script>
```

## Schritt 7: Verifizierung im Produktivbetrieb

Leeren Sie den PrestaShop-Cache und öffnen Sie Ihren Shop in einem Inkognito-Fenster. Prüfen Sie:

- Ergebnisse erscheinen während der Eingabe
- Tippfehlertoleranz funktioniert
- Produktbilder werden angezeigt
- Klick auf Ergebnis führt zur richtigen Produktseite

## Was als Nächstes kommt

- **Analytics**: Welche Anfragen ergeben keine Ergebnisse?
- **Synonyme**: Domänenspezifische Synonyme im Relevanz-Tab hinzufügen
- **Kurations**: Margenstarke Produkte für kommerzielle Anfragen priorisieren

Die Suche läuft. Jetzt beginnt die Optimierung für Conversion.
