Решение «строить vs покупать» для поисковой инфраструктуры редко сводится к простому сравнению ежемесячных цен подписок. Самостоятельный хостинг скрывает затраты на инженерное время, эксплуатацию и упущенную выгоду. Управляемые сервисы предлагают премиальное ценообразование, но включают обслуживание и надёжность.

В этой статье разбирается реальная совокупная стоимость владения (TCO) для обоих подходов на трёх типовых масштабах развёртывания.

## Категории затрат

Честное сравнение должно включать:

1. **Инфраструктура** — Серверы, хранилище, трафик, резервное копирование
2. **Эксплуатация** — Мониторинг, оповещения, реагирование на инциденты, обновления
3. **Инженерное время** — Первоначальная настройка, тюнинг, обслуживание, отладка
4. **Упущенная выгода** — Что ваша команда могла бы создавать вместо этого
5. **Затраты на масштабирование** — Стоимость добавления мощностей по мере роста

## Скрытые затраты самостоятельного хостинга

Помимо цифр, самостоятельный хостинг несёт риски, которые сложно оценить количественно, пока они не произойдут:

| Риск                        | Вероятность | Влияние                       | Стоимость смягчения        |
| --------------------------- | ----------- | ----------------------------- | -------------------------- |
| Отказ кластера              | Низкая      | Полный отказ поиска           | HA-настройка (2x инфра $)  |
| Повреждение данных          | Очень низк. | Восстановление из бэкапа      | RTO: 2-4 часа              |
| Сломанное обновление версии | Средняя     | Откат или исправление         | Staging-среда ($50/мес)    |
| Уязвимость безопасности     | Низкая      | Утечка данных                 | SLA по установке патчей    |
| Экстренное масштабирование  | Средняя     | Деградация производительности | Срочное выделение ресурсов |

Большинство команд недооценивают эти риски, принимая решение «строить vs покупать».

## Скрытые затраты управляемых сервисов

Управляемый поиск тоже не лишён рисков:

| Риск                   | Вероятность | Влияние             | Смягчение                        |
| ---------------------- | ----------- | ------------------- | -------------------------------- |
| Привязка к вендору     | Низкая      | Дорогой переход     | AACsearch: полный экспорт данных |
| Рост цены              | Средняя     | Удар по бюджету     | Зафиксировать годовую цену       |
| Отсутствие фич         | Низкая      | Нужны обходные пути | Оценить до перехода              |
| Задержка из-за региона | Низкая      | Медленные ответы    | Выбрать ближайший регион         |

AACsearch снижает риск привязки к вендору, будучи построенным на Typesense — вы можете экспортировать весь свой набор данных и запустить его на собственном кластере Typesense в любой момент.

## Анализ точки безубыточности

При каком масштабе самостоятельный хостинг становится дешевле управляемого?

```
Self-hosted фикс. затраты:  Настройка ($200-3K) + Ежемес. эксплуатация ($50-500)
Managed фикс. затраты:      Ежемесячная подписка ($0-249)

Точка безубыточности: Когда ежемесячная эксплуатация < премия подписки
```

Для большинства команд точка безубыточности никогда не достигается, потому что:

1. Инженерное время имеет стоимость роста — каждый час, потраченный на поисковую инфраструктуру, мог бы улучшить ваш продукт
2. Инфраструктурные затраты self-hosted растут с объёмом данных (больше серверов), а managed-затраты остаются плоскими
3. Риск инцидентов растёт с масштабом, а не снижается

## Нефинансовые факторы

Иногда самостоятельный хостинг имеет смысл даже при более высокой стоимости:

- **Комплаенс:** Данные никогда не должны покидать вашу инфраструктуру
- **Полный контроль:** Вам нужно кастомное поведение поиска, которое провайдер не поддерживает
- **Существующие инвестиции:** У вас уже есть команда и кластер Elasticsearch
- **Офлайн/air-gapped:** Ваше приложение работает в изолированной сети

Для всех остальных управляемый поиск обеспечивает лучшую окупаемость инвестиций.

## Структура принятия решений

```text
Поиск — ваше ключевое конкурентное преимущество?

└── Нет → Используйте управляемый сервис
        ├── < 50K документов → Бесплатный тариф (AACsearch / Algolia)
        ├── < 500K документов → Тариф Starter/Scale
        └── > 500K документов → Тариф Scale/Pro
```
