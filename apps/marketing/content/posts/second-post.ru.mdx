## Проблема релевантности, которая стоит вам продаж

Ваш самый популярный товар отображается на третьей странице. Покупатель ищет «кроссовки» и получает нулевые результаты, потому что в вашем каталоге используется «спортивная обувь». Кто-то вводит «дешёвый», и ваш поиск трактует это как ключевое слово вместо того, чтобы игнорировать.

Это проблемы релевантности. Они невидимы в аналитике до тех пор, пока кто-нибудь не сопоставит поиски без результатов с брошенными корзинами — и когда это происходит, цифры всегда оказываются хуже ожидаемого.

Хорошая новость: проблемы релевантности решаемы без команды специалистов по данным. Три инструмента закрывают 80% из них: **синонимы**, **курации** и **стоп-слова**.

## Синонимы: когда покупатели выражаются по-другому

Правило синонимов говорит поисковому движку считать два или более терминов эквивалентными. Их два вида:

### Эквивалентные синонимы (двунаправленные)

```json
{
	"synonyms": ["кроссовки", "спортивная обувь", "кеды", "тренировочные туфли"]
}
```

Когда покупатель ищет «кеды», результаты для «кроссовок» и «спортивной обуви» появляются рядом с буквальными совпадениями для «кеды». Обратное тоже верно.

**Используйте для:** региональных различий в словарном запасе, вариаций торговых марок, профессионального жаргона в сравнении с потребительскими терминами.

### Однонаправленные синонимы

```json
{
	"root": "ноутбук",
	"synonyms": ["портативный компьютер", "лэптоп"]
}
```

Поиск по «ноутбук» возвращает результаты, соответствующие «лэптоп», — но поиск по «лэптоп» НЕ возвращает результаты для «ноутбук». Это полезно, когда один термин является авторитетным, а остальные — крайними случаями.

**Используйте для:** технических терминов, соответствующих канонической категории товаров; устаревших названий товаров, перенаправляемых на актуальные.

### Измерение влияния синонимов

В AACsearch Analytics фильтруйте по запросам, к которым добавили синонимы. Сравните долю поисков без результатов до и после внедрения. Хорошо расположенный набор синонимов обычно снижает поиски без результатов на 15–40% для целевых терминов.

## Курации: ручное управление, когда алгоритмов недостаточно

Курации позволяют закреплять конкретные товары в топе результатов поиска или исключать товары, которые не должны появляться для определённого запроса.

### Закрепление товара

Ваша летняя распродажа имеет акционный товар. Вы хотите, чтобы он выходил первым по запросам «сандалии», «летняя обувь» и «пляж»:

```json
{
	"id": "curation-summer-sale",
	"query": "сандалии",
	"includedHits": [{ "id": "product-1234", "position": 1 }]
}
```

Товар 1234 отображается на позиции 1 независимо от оценки релевантности. Всё остальное ранжируется нормально ниже.

### Исключение товаров

Товар закончился на складе, но вы ещё не удалили его из индекса. Или он снят с производства, и вы не хотите, чтобы он появлялся по определённым запросам:

```json
{
	"query": "беговые кроссовки",
	"excludedHits": ["product-5678", "product-9012"]
}
```

### Когда использовать курации

Курации — это скальпель, а не кисть. Оставляйте их для:

- **Сезонных акций** — закрепляйте товары в продаже по релевантным запросам во время кампании
- **Запуска новых товаров** — обеспечивайте видимость для позиций, у которых ещё нет рейтинговых сигналов
- **Отзывов или правовых ограничений** — удаляйте конкретные товары из результатов по определённым поискам

Не используйте курации для компенсации плохой индексации. Если 50 товаров требуют ручного закрепления, основная проблема, вероятно, в схеме или весовых коэффициентах — исправьте это вместо этого.

## Стоп-слова: игнорирование слов, создающих шум

Стоп-слова — это термины, которые поисковый движок полностью игнорирует. Распространённые русские стоп-слова — «в», «на», «из», «и» — обычно обрабатываются автоматически. Сайты e-commerce нуждаются в пользовательских стоп-словах для своего специфического контекста.

### Распространённые стоп-слова в e-commerce

```json
{
	"stopwords": ["дёшево", "купить", "приобрести", "найти", "лучший", "хороший", "заказать"]
}
```

Когда покупатель ищет «купить дешёвые беговые кроссовки», поисковый движок обрабатывает «беговые кроссовки» — то, что им действительно нужно. Без стоп-слов «купить» и «дёшево» становятся ключевыми словами, которые могут снизить оценку релевантности.

### Что НЕ нужно добавлять как стоп-слово

Будьте осторожны с терминами, связанными с ценой. «Дёшево» — кандидат в стоп-слова, потому что это намерение, а не данные о товаре. «Скидка» — сложнее: у некоторых ваших товаров буквально может быть слово «скидка» в названии или описании. Тестируйте перед внедрением.

### Антипаттерн стоп-слов

Некоторые команды добавляют названия конкурирующих брендов как стоп-слова, чтобы не показывать их товары, когда покупатели ищут конкурента. Это ошибка — покупатели, ищущие конкурента, являются возможностью для конверсии, а не проблемой. Покажите им свою лучшую альтернативу.

## Всё вместе: аудит релевантности

Проводите этот аудит ежеквартально:

1. **Экспортируйте топ-100 запросов с нулевыми результатами.** Это ваши приоритетные кандидаты для синонимов.
2. **Проверьте топ-100 запросов с CTR ниже 5%.** Низкий CTR при высоком объёме означает, что первыми ранжируются неподходящие товары — кандидаты для кураций.
3. **Проанализируйте запросы, где покупатели добавляют «дёшево», «купить» или «найти».** Добавьте их в список стоп-слов.
4. **A/B-тестируйте наборы синонимов**, сначала развёртывая их на одном регионе или сегменте трафика.

## Начинайте с малого, измеряйте всё

Ловушка в настройке релевантности — делать слишком много сразу и не понимать, что именно сдвинуло показатели. Добавьте один набор синонимов, измерьте в течение недели, затем добавьте ещё один. Курации имеют немедленный эффект — вы увидите изменение CTR в течение 24 часов после внедрения.

Релевантность никогда не бывает «готовой». Новые товары вводят новый словарный запас. Сезонные тенденции меняют то, что ищут покупатели. Ежеквартальный обзор позволяет опережать отклонения.
