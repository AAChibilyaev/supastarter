Ce guide vous emmène d'un compte AACsearch vierge à une expérience de recherche en direct sur votre site. Que vous ajoutiez la recherche à une boutique e-commerce, un site de documentation ou une application SaaS, chaque étape est couverte.

## Aperçu : Le Parcours en 7 Étapes

1. Créez votre compte et votre premier index
2. Préparez vos données
3. Importez les documents
4. Configurez le comportement de recherche
5. Choisissez votre chemin d'intégration
6. Déployez en production
7. Surveillez et améliorez

Temps total : 20 à 30 minutes. La majeure partie concerne la préparation des données — la configuration technique prend moins de 5 minutes.

## Étape 1 : Créez Votre Compte

Rendez-vous sur [aacsearch.com/signup](/signup) et créez un compte gratuit. Le niveau gratuit comprend :

- 1 index
- 10 000 documents
- Opérations de recherche illimitées
- Analytics intégrés
- Support communautaire

Aucune carte de crédit requise. Vous arriverez sur le Dashboard — votre centre de commande pour toutes les opérations de recherche.

## Étape 2 : Créez Votre Premier Index

Cliquez sur **New Index** dans le Dashboard. Vous aurez besoin de :
- **Nom d'index :** `products`, `docs`, `blog` — adaptez à votre type de données
- **Champ de tri par défaut :** Généralement `_text_match` (pertinence) ou un champ numérique comme `price`

L'index est provisionné immédiatement. Typesense crée la collection sous-jacente, et vos clés API sont générées automatiquement.

**Comprendre les Clés API :**
- **Clé recherche uniquement** (`ss_search_*`) : Pour utilisation navigateur/côté client — peut chercher mais pas modifier les données
- **Clé admin** (`ss_admin_*`) : Pour opérations côté serveur — peut créer/mettre à jour/supprimer des documents
- **Clé limitée** (`ss_scoped_*`) : Pour configurations multi-tenant — restreinte à des conditions de filtre spécifiques

Stockez votre clé admin dans les variables d'environnement. La clé de recherche peut être publique (elle est en lecture seule par conception).

## Étape 3 : Préparez Vos Données

AACsearch fonctionne avec des documents plats et dénormalisés. Chaque document est un objet JSON d'une taille maximale de 100 Ko.

**Exemple de produit e-commerce :**
```json
{
  "id": "prod_001",
  "name": "Wireless Bluetooth Headphones",
  "description": "Noise-cancelling wireless headphones with 30-hour battery life",
  "price": 79.99,
  "brand": "SoundMax",
  "category": "Electronics",
  "image_url": "https://example.com/images/headphones.jpg",
  "in_stock": true,
  "rating": 4.5,
  "tags": ["wireless", "audio", "bluetooth"]
}
```

**Règles clés :**
- Chaque document nécessite un champ `id` unique (chaîne de caractères)
- Les tableaux sont pris en charge (`tags`, `colors`, `sizes`) — ils deviennent automatiquement des facettes multi-valeurs
- L'imbrication est limitée — aplatissez les données connexes dans le document
- Les noms de champs utilisent `snake_case` (convention Typesense)

## Étape 4 : Importez Vos Documents

### Option A : Importation via le Dashboard (Idéal pour la première configuration)

1. Accédez à votre index → **Import**
2. Téléchargez un fichier JSON ou CSV (jusqu'à 100 Mo par import)
3. AACsearch détecte automatiquement les types de champs (chaîne, nombre, booléen, tableau)
4. Vérifiez le schéma détecté et effectuez des ajustements
5. Cliquez sur **Import** — les documents sont indexés instantanément

### Option B : Importation via API (Idéal pour la synchronisation continue)

```bash
curl -X POST "https://api.aacsearch.com/indices/products/documents/import" \
  -H "X-API-KEY: ss_admin_your_key" \
  -H "Content-Type: application/json" \
  -d '[
    {"id": "1", "name": "Product 1", "price": 29.99},
    {"id": "2", "name": "Product 2", "price": 49.99}
  ]'
```

Taille de lot : 10 000 documents par requête est optimal. Des lots plus grands peuvent expirer ; des lots plus petits conviennent pour les mises à jour en temps réel.

### Option C : Connecteur (Idéal pour les plateformes CMS)

Si vous utilisez PrestaShop, Bitrix24, WooCommerce ou Shopify, utilisez le connecteur pré-construit :

```bash
# Configuration du connecteur PrestaShop
composer require aacsearch/prestashop-connector
php bin/console aacsearch:sync --store=my-store.com
```

Le connecteur gère le mappage de schéma, les mises à jour incrémentielles et la synchronisation des suppressions.

## Étape 5 : Configurez le Comportement de Recherche

### Champs Recherchables

Dans le Dashboard, marquez les champs qui sont recherchables :

| Champ       | Recherchable | Boost | Facette | Triable |
| ----------- | ------------ | ----- | ------- | ------- |
| name        | Oui          | 3x    | Non     | Non     |
| description | Oui          | 1x    | Non     | Non     |
| brand       | Oui          | 2x    | Oui     | Oui     |
| category    | Non          | —     | Oui     | Non     |
| price       | Non          | —     | Oui     | Oui     |

Le boost contrôle le classement : une correspondance dans `name` (boost 3x) est classée plus haut qu'une correspondance dans `description` (boost 1x).

### Tolérance aux Fautes de Frappe

Configurez par index :
- **Fautes max :** 1 (par défaut) ou 2 pour les champs courts comme les codes SKU
- **Boost de correspondance exacte :** Les correspondances exactes sont classées 2x plus haut que les correspondances corrigées
- **Correspondance de préfixe :** Les requêtes correspondent aux préfixes (gère la saisie partielle)

### Groupes de Synonymes

Créez des groupes de synonymes pour la terminologie spécifique à votre secteur :

```
["sneakers", "trainers", "athletic shoes", "running shoes"]
["laptop", "notebook", "macbook", "ultrabook"]
["sofa", "couch", "settee", "loveseat"]
```

Chaque groupe est une liste de termes équivalents. Une recherche pour « trainers » retourne des résultats correspondant à « sneakers » et vice versa.

## Étape 6 : Choisissez Votre Chemin d'Intégration

### Chemin A : Widget (5 minutes, zéro code)

Le Widget AACsearch est une balise script à intégrer. Idéal pour les sites de documentation, les blogs et les boutiques e-commerce simples.

```html
<script src="https://cdn.aacsearch.com/widget.js"></script>
<div id="search"></div>
<script>
  new AACsearchWidget({
    apiKey: 'ss_search_your_key',
    index: 'products',
    mountElement: '#search'
  });
</script>
```

### Chemin B : API REST (30 minutes, contrôle total)

Utilisez l'API REST pour les frontends personnalisés, les applications mobiles ou la recherche côté serveur.

```javascript
const response = await fetch('https://api.aacsearch.com/indices/products/search', {
  method: 'POST',
  headers: {
    'X-API-KEY': 'ss_search_your_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    q: 'wireless headphones',
    query_by: 'name,description',
    facet_by: 'brand,category',
    max_facet_values: 10
  })
});

const results = await response.json();
```

### Chemin C : SDK Côté Serveur (15 minutes, type-safe)

Disponible pour Node.js, Python, PHP, Go et Ruby.

```typescript
// npm install @aacsearch/client
import { AACSearchClient } from '@aacsearch/client';

const client = new AACSearchClient({
  apiKey: process.env.AACSEARCH_ADMIN_KEY,
});

const results = await client.search('products', {
  q: 'wireless headphones',
  search_fields: ['name', 'description'],
  facet_fields: ['brand', 'category'],
});
```

### Chemin D : Connecteur Pré-construit (10 minutes, spécifique CMS)

- **PrestaShop :** `composer require aacsearch/prestashop-connector`
- **Bitrix24 :** Installez depuis le Bitrix Marketplace
- **WooCommerce :** `wp plugin install aacsearch-woocommerce`
- **Shopify :** Recherchez AACsearch dans l'App Store Shopify

## Étape 7 : Déployez en Production

### Liste de Vérification Pré-Lancement

- [ ] Les clés API utilisent des variables d'environnement (pas en dur)
- [ ] Les clés recherche uniquement sont utilisées côté client (jamais les clés admin)
- [ ] La limitation de débit est configurée par locataire (si multi-tenant)
- [ ] CORS configuré pour votre domaine
- [ ] Le suivi analytics est activé
- [ ] Les requêtes sans résultat sont enregistrées pour l'analyse des lacunes de contenu
- [ ] Le Widget est thématisé pour correspondre à votre marque (variables CSS)
- [ ] La mise en page mobile est testée (tiroir de facettes, fiches résultats)

### Mise en Production

1. Basculez de votre index de test à votre index de production
2. Surveillez la latence de recherche dans les analytics du Dashboard
3. Vérifiez les rapports de requêtes sans résultat — ce sont des opportunités de contenu
4. Mettez en place un examen hebdomadaire des analytics de recherche

## Dépannage des Problèmes Courants

| Problème                          | Cause probable                        | Correctif                                             |
| --------------------------------- | ------------------------------------- | ----------------------------------------------------- |
| Aucun résultat pour toute requête | Documents non encore indexés          | Vérifiez le nombre de documents dans le Dashboard → réimportez |
| Tolérance aux fautes inactive     | Champ non marqué comme recherchable   | Marquez le champ comme recherchable dans les paramètres du schéma |
| Le Widget ne s'affiche pas        | Lien CSS manquant ou ID div incorrect | Vérifiez que le CSS est chargé et que l'ID mountElement correspond |
| Recherche lente (>200 ms)         | Trop de champs recherchables          | Limitez les champs recherchables aux 3-5 plus importants |
| Valeurs de facettes absentes      | Champ non marqué comme facette        | Activez la facette dans les paramètres du schéma       |
| Données inter-locataires visibles | Pas de clé API limitée                | Créez des clés limitées avec `token_filter` par locataire |
| L'API retourne 401                | Clé API expirée ou invalide           | Régénérez la clé depuis le Dashboard                   |
| L'import échoue avec timeout      | Lot trop volumineux                   | Divisez en lots de 10 000 documents                   |
