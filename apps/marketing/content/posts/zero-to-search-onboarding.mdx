This guide takes you from a blank AACsearch account to a live search experience on your site. Whether you're adding search to an e-commerce store, documentation site, or SaaS app, every step is covered.

## Overview: The 7-Step Journey

1. Create your account and first index
2. Prepare your data
3. Import documents
4. Configure search behavior
5. Choose your integration path
6. Deploy to production
7. Monitor and improve

Total time: 20-30 minutes. Most of it is data preparation — the technical setup takes under 5 minutes.

## Step 1: Create Your Account

Go to [aacsearch.com/signup](/signup) and create a free account. The free tier includes:

- 1 index
- 10,000 documents
- Unlimited search operations
- Built-in analytics
- Community support

No credit card required. You'll land on the dashboard — your command center for all search operations.

## Step 2: Create Your First Index

Click **New Index** in the dashboard. You'll need:

- **Index name:** `products`, `docs`, `blog` — match your data type
- **Default sorting field:** Typically `_text_match` (relevance) or a numerical field like `price`

The index is provisioned immediately. Typesense creates the underlying collection, and your API keys are generated automatically.

**Understanding API Keys:**

- **Search-only key** (`ss_search_*`): For browser/client-side use — can search but not modify data
- **Admin key** (`ss_admin_*`): For server-side operations — can create/update/delete documents
- **Scoped key** (`ss_scoped_*`): For multi-tenant setups — restricted to specific filter conditions

Store your admin key in environment variables. The search key can be public (it's read-only by design).

## Step 3: Prepare Your Data

AACsearch works with flat, denormalized documents. Each document is a JSON object with a maximum size of 100KB.

**E-commerce product example:**

```json
{
	"id": "prod_001",
	"name": "Wireless Bluetooth Headphones",
	"description": "Noise-cancelling wireless headphones with 30-hour battery life",
	"price": 79.99,
	"brand": "SoundMax",
	"category": "Electronics",
	"image_url": "https://example.com/images/headphones.jpg",
	"in_stock": true,
	"rating": 4.5,
	"tags": ["wireless", "audio", "bluetooth"]
}
```

**Key rules:**

- Every document needs a unique `id` field (string)
- Arrays are supported (`tags`, `colors`, `sizes`) — they auto-become multi-value facets
- Nesting is limited — flatten related data into the document
- Field names use `snake_case` (Typesense convention)

## Step 4: Import Your Documents

### Option A: Dashboard Upload (Best for first-time setup)

1. Navigate to your index → **Import**
2. Upload a JSON or CSV file (up to 100MB per upload)
3. AACsearch auto-detects field types (string, number, boolean, array)
4. Review the detected schema and make adjustments
5. Click **Import** — documents are indexed instantly

### Option B: API Import (Best for ongoing sync)

```bash
curl -X POST "https://api.aacsearch.com/indices/products/documents/import" \
  -H "X-API-KEY: ss_admin_your_key" \
  -H "Content-Type: application/json" \
  -d '[
    {"id": "1", "name": "Product 1", "price": 29.99},
    {"id": "2", "name": "Product 2", "price": 49.99}
  ]'
```

Batch size: 10,000 documents per request is optimal. Larger batches may timeout; smaller batches are fine for real-time updates.

### Option C: Connector (Best for CMS platforms)

If you're using PrestaShop, Bitrix24, WooCommerce, or Shopify, use the pre-built connector:

```bash
# PrestaShop connector setup
composer require aacsearch/prestashop-connector
php bin/console aacsearch:sync --store=my-store.com
```

The connector handles schema mapping, incremental updates, and delete synchronization.

## Step 5: Configure Search Behavior

### Searchable Fields

In the dashboard, mark which fields are searchable:

| Field       | Searchable | Boost | Facet | Sortable |
| ----------- | ---------- | ----- | ----- | -------- |
| name        | Yes        | 3x    | No    | No       |
| description | Yes        | 1x    | No    | No       |
| brand       | Yes        | 2x    | Yes   | Yes      |
| category    | No         | —     | Yes   | No       |
| price       | No         | —     | Yes   | Yes      |

Boost controls ranking: a match in `name` (boost 3x) ranks higher than a match in `description` (boost 1x).

### Typo Tolerance

Configure per-index:

- **Max typos:** 1 (default) or 2 for short fields like SKU codes
- **Exact match boost:** Exact matches rank 2x higher than typo-corrected matches
- **Prefix matching:** Queries match prefixes (handles partial typing)

### Synonym Groups

Create synonym groups for industry-specific terminology:

```
["sneakers", "trainers", "athletic shoes", "running shoes"]
["laptop", "notebook", "macbook", "ultrabook"]
["sofa", "couch", "settee", "loveseat"]
```

Each group is a list of equivalent terms. A search for "trainers" returns results matching "sneakers" and vice versa.

## Step 6: Choose Your Integration Path

### Path A: Widget (5 minutes, zero code)

The AACsearch widget is a drop-in script tag. Best for documentation sites, blogs, and simple e-commerce stores.

```html
<script src="https://cdn.aacsearch.com/widget.js"></script>
<div id="search"></div>
<script>
	new AACsearchWidget({
		apiKey: "ss_search_your_key",
		index: "products",
		mountElement: "#search",
	});
</script>
```

### Path B: REST API (30 minutes, full control)

Use the REST API for custom frontends, mobile apps, or server-side search.

```javascript
const response = await fetch("https://api.aacsearch.com/indices/products/search", {
	method: "POST",
	headers: {
		"X-API-KEY": "ss_search_your_key",
		"Content-Type": "application/json",
	},
	body: JSON.stringify({
		q: "wireless headphones",
		query_by: "name,description",
		facet_by: "brand,category",
		max_facet_values: 10,
	}),
});

const results = await response.json();
```

### Path C: Server-side SDK (15 minutes, type-safe)

Available for Node.js, Python, PHP, Go, and Ruby.

```typescript
// npm install @aacsearch/client
import { AACSearchClient } from "@aacsearch/client";

const client = new AACSearchClient({
	apiKey: process.env.AACSEARCH_ADMIN_KEY,
});

const results = await client.search("products", {
	q: "wireless headphones",
	search_fields: ["name", "description"],
	facet_fields: ["brand", "category"],
});
```

### Path D: Pre-built Connector (10 minutes, CMS-specific)

- **PrestaShop:** `composer require aacsearch/prestashop-connector`
- **Bitrix24:** Install from Bitrix Marketplace
- **WooCommerce:** `wp plugin install aacsearch-woocommerce`
- **Shopify:** Search AACsearch in Shopify App Store

## Step 7: Deploy to Production

### Pre-Launch Checklist

- [ ] API keys are using environment variables (not hardcoded)
- [ ] Search-only keys used on the client (never admin keys)
- [ ] Rate limiting configured per tenant (if multi-tenant)
- [ ] CORS configured for your domain
- [ ] Analytics tracking enabled
- [ ] Zero-result queries being logged for content gap analysis
- [ ] Widget themed to match your brand (CSS variables)
- [ ] Mobile layout tested (facet drawer, result cards)

### Go Live

1. Switch from your test index to your production index
2. Monitor search latency in the dashboard analytics
3. Check zero-result query reports — these are content opportunities
4. Set up a weekly search analytics review

## Troubleshooting Common Issues

| Problem                    | Likely Cause                        | Fix                                               |
| -------------------------- | ----------------------------------- | ------------------------------------------------- |
| No results for any query   | Documents not indexed yet           | Check document count in dashboard → re-import     |
| Typo tolerance not working | Field not marked as searchable      | Mark field as searchable in schema settings       |
| Widget not rendering       | Missing CSS link or div ID mismatch | Verify CSS is loaded and mountElement ID matches  |
| Slow search (>200ms)       | Too many searchable fields          | Limit searchable fields to 3-5 most important     |
| Facet values not showing   | Field not marked as facet           | Enable faceting in schema settings                |
| Cross-tenant data visible  | No scoped API key                   | Create scoped keys with `token_filter` per tenant |
| API returning 401          | Expired or invalid API key          | Regenerate key from dashboard                     |
| Import fails with timeout  | Batch too large                     | Split into batches of 10,000 documents            |
