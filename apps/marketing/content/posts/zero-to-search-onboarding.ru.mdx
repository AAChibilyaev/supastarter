Это руководство проведёт вас от пустого аккаунта AACsearch до работающего поиска на вашем сайте. Добавляете ли вы поиск в интернет-магазин, сайт документации или SaaS-приложение — каждый шаг описан.

## Обзор: путь из 7 шагов

1. Создание аккаунта и первого индекса
2. Подготовка данных
3. Импорт документов
4. Настройка поведения поиска
5. Выбор пути интеграции
6. Развёртывание в продакшене
7. Мониторинг и улучшение

Общее время: 20–30 минут. Большая часть уходит на подготовку данных — техническая настройка занимает менее 5 минут.

## Шаг 1: Создание аккаунта

Перейдите на [aacsearch.com/signup](/signup) и создайте бесплатный аккаунт. Бесплатный тариф включает:

- 1 индекс
- 10 000 документов
- Безлимитные поисковые операции
- Встроенную аналитику
- Поддержку сообщества

Никакой кредитной карты не требуется. Вы попадёте на панель управления — ваш командный центр для всех поисковых операций.

## Шаг 2: Создание первого индекса

Нажмите **New Index** в панели управления. Вам потребуется:

- **Имя индекса:** `products`, `docs`, `blog` — соответствует типу ваших данных
- **Поле сортировки по умолчанию:** Обычно `_text_match` (релевантность) или числовое поле, например `price`

Индекс создаётся мгновенно. Typesense формирует базовую коллекцию, и ваши API-ключи генерируются автоматически.

**Понимание API-ключей:**

- **Ключ только для поиска** (`ss_search_*`): Для использования в браузере/на клиенте — может искать, но не изменять данные
- **Админ-ключ** (`ss_admin_*`): Для серверных операций — может создавать/обновлять/удалять документы
- **Ограниченный ключ** (`ss_scoped_*`): Для мультитенантных конфигураций — ограничен определёнными условиями фильтрации

Храните админ-ключ в переменных окружения. Поисковый ключ может быть публичным (он по замыслу только для чтения).

## Шаг 3: Подготовка данных

AACsearch работает с плоскими, денормализованными документами. Каждый документ — это JSON-объект максимальным размером 100KB.

**Пример товара интернет-магазина:**

```json
{
	"id": "prod_001",
	"name": "Wireless Bluetooth Headphones",
	"description": "Noise-cancelling wireless headphones with 30-hour battery life",
	"price": 79.99,
	"brand": "SoundMax",
	"category": "Electronics",
	"image_url": "https://example.com/images/headphones.jpg",
	"in_stock": true,
	"rating": 4.5,
	"tags": ["wireless", "audio", "bluetooth"]
}
```

**Ключевые правила:**

- Каждый документ должен иметь уникальное поле `id` (строка)
- Массивы поддерживаются (`tags`, `colors`, `sizes`) — они автоматически становятся многозначными фасетами
- Вложенность ограничена — уплощайте связанные данные в рамках одного документа
- Имена полей используйте в `snake_case` (соглашение Typesense)

## Шаг 4: Импорт документов

### Вариант A: Загрузка через панель управления (лучше для первого запуска)

1. Перейдите к вашему индексу → **Import**
2. Загрузите JSON- или CSV-файл (до 100MB за одну загрузку)
3. AACsearch автоматически определяет типы полей (строка, число, булево, массив)
4. Проверьте обнаруженную схему и внесите корректировки
5. Нажмите **Import** — документы индексируются мгновенно

### Вариант B: Импорт через API (лучше для постоянной синхронизации)

```bash
curl -X POST "https://api.aacsearch.com/indices/products/documents/import" \
  -H "X-API-KEY: ss_admin_your_key" \
  -H "Content-Type: application/json" \
  -d '[
    {"id": "1", "name": "Product 1", "price": 29.99},
    {"id": "2", "name": "Product 2", "price": 49.99}
  ]'
```

Оптимальный размер пакета: 10 000 документов на запрос. Более крупные пакеты могут привести к тайм-ауту; меньшие пакеты подходят для обновлений в реальном времени.

### Вариант C: Коннектор (лучше для CMS-платформ)

Если вы используете PrestaShop, Bitrix24, WooCommerce или Shopify, воспользуйтесь готовым коннектором:

```bash
# Настройка коннектора PrestaShop
composer require aacsearch/prestashop-connector
php bin/console aacsearch:sync --store=my-store.com
```

Коннектор обрабатывает сопоставление схем, инкрементальные обновления и синхронизацию удалений.

## Шаг 5: Настройка поведения поиска

### Поисковые поля

В панели управления отметьте, какие поля доступны для поиска:

| Поле        | Поиск | Повышение | Фасет | Сортировка |
| ----------- | ----- | --------- | ----- | ---------- |
| name        | Да    | 3x        | Нет   | Нет        |
| description | Да    | 1x        | Нет   | Нет        |
| brand       | Да    | 2x        | Да    | Да         |
| category    | Нет   | —         | Да    | Нет        |
| price       | Нет   | —         | Да    | Да         |

Повышение управляет ранжированием: совпадение в `name` (повышение 3x) ранжируется выше, чем совпадение в `description` (повышение 1x).

### Устойчивость к опечаткам

Настройка для каждого индекса:

- **Макс. опечаток:** 1 (по умолчанию) или 2 для коротких полей, таких как коды SKU
- **Повышение точного совпадения:** Точные совпадения ранжируются в 2x выше, чем исправленные по опечаткам
- **Префиксное совпадение:** Запросы сопоставляются с префиксами (обрабатывает частичный ввод)

### Группы синонимов

Создавайте группы синонимов для отраслевой терминологии:

```
["sneakers", "trainers", "athletic shoes", "running shoes"]
["laptop", "notebook", "macbook", "ultrabook"]
["sofa", "couch", "settee", "loveseat"]
```

Каждая группа — это список эквивалентных терминов. Поиск «trainers» возвращает результаты, соответствующие «sneakers», и наоборот.

## Шаг 6: Выбор пути интеграции

### Путь A: Widget (5 минут, ноль кода)

Widget AACsearch — это подключаемый тег `<script>`. Лучше всего подходит для сайтов документации, блогов и простых интернет-магазинов.

```html
<script src="https://cdn.aacsearch.com/widget.js"></script>
<div id="search"></div>
<script>
	new AACsearchWidget({
		apiKey: "ss_search_your_key",
		index: "products",
		mountElement: "#search",
	});
</script>
```

### Путь B: REST API (30 минут, полный контроль)

Используйте REST API для кастомных фронтендов, мобильных приложений или серверного поиска.

```javascript
const response = await fetch("https://api.aacsearch.com/indices/products/search", {
	method: "POST",
	headers: {
		"X-API-KEY": "ss_search_your_key",
		"Content-Type": "application/json",
	},
	body: JSON.stringify({
		q: "wireless headphones",
		query_by: "name,description",
		facet_by: "brand,category",
		max_facet_values: 10,
	}),
});

const results = await response.json();
```

### Путь C: Серверный SDK (15 минут, типобезопасный)

Доступен для Node.js, Python, PHP, Go и Ruby.

```typescript
// npm install @aacsearch/client
import { AACSearchClient } from "@aacsearch/client";

const client = new AACSearchClient({
	apiKey: process.env.AACSEARCH_ADMIN_KEY,
});

const results = await client.search("products", {
	q: "wireless headphones",
	search_fields: ["name", "description"],
	facet_fields: ["brand", "category"],
});
```

### Путь D: Готовый коннектор (10 минут, для конкретных CMS)

- **PrestaShop:** `composer require aacsearch/prestashop-connector`
- **Bitrix24:** Установка из Bitrix Marketplace
- **WooCommerce:** `wp plugin install aacsearch-woocommerce`
- **Shopify:** Поиск AACsearch в Shopify App Store

## Шаг 7: Развёртывание в продакшене

### Контрольный список перед запуском

- [ ] API-ключи используют переменные окружения (не хардкод)
- [ ] На клиенте используются ключи только для поиска (никогда админ-ключи)
- [ ] Настроено ограничение скорости запросов для каждого тенанта (если мультитенантно)
- [ ] CORS настроен для вашего домена
- [ ] Включено отслеживание аналитики
- [ ] Запросы без результатов логируются для анализа пробелов в контенте
- [ ] Widget стилизован под ваш бренд (CSS-переменные)
- [ ] Проверен мобильный макет (панель фасетов, карточки результатов)

### Запуск

1. Переключитесь с тестового индекса на продакшен-индекс
2. Отслеживайте задержки поиска в аналитике панели управления
3. Проверяйте отчёты по запросам без результатов — это возможности для контента
4. Настройте еженедельный обзор аналитики поиска

## Устранение распространённых проблем

| Проблема                             | Вероятная причина                      | Решение                                                     |
| ------------------------------------ | -------------------------------------- | ----------------------------------------------------------- |
| Нет результатов ни по одному запросу | Документы ещё не проиндексированы      | Проверьте количество документов в панели → переимпортируйте |
| Устойчивость к опечаткам не работает | Поле не отмечено как поисковое         | Отметьте поле как поисковое в настройках схемы              |
| Widget не отображается               | Отсутствует CSS-ссылка или неверный ID | Проверьте, что CSS загружен и mountElement ID совпадает     |
| Медленный поиск (>200 мс)            | Слишком много поисковых полей          | Ограничьте поисковые поля 3–5 самыми важными                |
| Значения фасетов не отображаются     | Поле не отмечено как фасет             | Включите фасетирование в настройках схемы                   |
| Видны данные других тенантов         | Нет ограниченного API-ключа            | Создайте ограниченные ключи с `token_filter` на тенант      |
| API возвращает 401                   | Истёкший или недействительный API-ключ | Сгенерируйте новый ключ из панели управления                |
| Импорт завершается с тайм-аутом      | Слишком большой пакет                  | Разделите на пакеты по 10 000 документов                    |
